# Pattern Cheat Sheet

## Creational Patterns

| Pattern | Purpose | Use Case |
|---------|---------|----------|
| **Singleton** | One instance globally | Config, Logger |
| **Factory** | Create without specifying class | Object creation logic |
| **Abstract Factory** | Family of related objects | UI themes, cross-platform |
| **Builder** | Complex object construction | Fluent API, immutable objects |
| **Prototype** | Clone existing objects | Caching, undo functionality |

## Structural Patterns

| Pattern | Purpose | Use Case |
|---------|---------|----------|
| **Adapter** | Convert interface | Legacy integration |
| **Bridge** | Decouple abstraction from implementation | Platform-independent |
| **Composite** | Tree structures | UI components, file systems |
| **Decorator** | Add behavior dynamically | Logging, caching |
| **Facade** | Simplified interface | Library wrapper |
| **Flyweight** | Share common state | Large number of objects |

## Behavioral Patterns

| Pattern | Purpose | Use Case |
|---------|---------|----------|
| **Observer** | Event notification | GUI, pub/sub |
| **Strategy** | Interchangeable algorithms | Sorting, validation |
| **Command** | Encapsulate request | Undo/redo, queuing |
| **State** | Object behavior changes with state | FSM |
| **Template Method** | Define algorithm skeleton | Frameworks |
| **Iterator** | Traverse collection | Collection access |
| **Chain of Responsibility** | Multiple handlers | Request processing |
