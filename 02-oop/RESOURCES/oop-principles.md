# OOP Principles

## The Four Pillars

### 1. Encapsulation
Bundling data and methods that operate on that data within a single unit (class). Hides internal state and requires all interaction through an object's methods.

**Benefits:**
- Data hiding and protection
- Improved modularity
- Easier maintenance

### 2. Inheritance
A mechanism where a new class inherits properties and behaviors from an existing class.

**Benefits:**
- Code reusability
- Established relationships between classes
- Logical hierarchy

### 3. Polymorphism
The ability of objects to take on multiple forms. Achieved through method overriding and overloading.

**Benefits:**
- Flexibility
- Extensibility
- Interface-based programming

### 4. Abstraction
Hiding complex implementation details and showing only the necessary features of an object.

**Benefits:**
- Reduced complexity
- Focus on what object does vs how it does it
- Improved maintainability
