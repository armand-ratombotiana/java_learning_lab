# Why OOP Exists

## Purpose

OOP was created to solve software complexity problems. As programs grew larger, procedural code became difficult to maintain. OOP provides a way to organize code that mirrors real-world entities.

## Historical Context

- **1967**: Simula - first OOP language (created by Ole-Johan Dahl)
- **1970s**: Smalltalk - pure OOP, influenced Java
- **1991**: Oak (Java's predecessor) - embedded in consumer devices
- **1995**: Java - "write once, run anywhere"

## Why OOP Matters

1. **Modularity**: Code organized into objects
2. **Reusability**: Inheritance enables code reuse
3. **Maintainability**: Encapsulation hides complexity
4. **Extensibility**: Polymorphism allows easy extension
5. **Real-world modeling**: Objects represent real entities
