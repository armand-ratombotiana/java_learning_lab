# Mental Models for OOP

## Class as Blueprint

```
┌─────────────────────────────────────┐
│           CLASS                     │
├─────────────────────────────────────┤
│  Fields (Data)                      │
│  ┌───────────────────────────────┐  │
│  │ String name                  │  │
│  │ int age                      │  │
│  └───────────────────────────────┘  │
├─────────────────────────────────────┤
│  Methods (Behavior)                 │
│  ┌───────────────────────────────┐  │
│  │ void speak()                  │  │
│  │ void walk()                   │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
            │
            ▼
┌─────────────────────────────────────┐
│         OBJECT (Instance)           │
│  name = "John", age = 25            │
└─────────────────────────────────────┘
```

## Encapsulation = Data + Methods together

```
┌─────────────────────────────────────┐
│         Encapsulation               │
│  ┌─────────────────────────────┐    │
│  │ private data                │    │
│  │ - balance                   │    │
│  └─────────────────────────────┘    │
│  ┌─────────────────────────────┐    │
│  │ public methods              │    │
│  │ + deposit()                 │    │
│  │ + withdraw()                │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```
