## 💡 Key Success Factors

### Before Interview
1. ✅ Know the company's core products
2. ✅ Understand their scale (users, QPS, data)
3. ✅ Practice the problems (multiple times)
4. ✅ Time yourself strictly (no cheating)
5. ✅ Get comfortable on whiteboard

1. ✅ Ask clarifying questions first
2. ✅ Discuss approach before coding
3. ✅ Code cleanly and methodically
4. ✅ Explain while you code

6. ✅ Discuss complexity tradeoffs
7. ✅ Suggest improvements/optimizations

### After Interview
1. ✅ Thank the interviewer

3. ✅ Note what went well and poorly
4. ✅ Plan improvements for next round
5. ✅ Never blame the environment

## 📞 Still Have Questions?

### Common Questions
**Q: Which company should I apply to first?**  
A: Start with companies you're most interested in. But practice with easier interview pools first.

**Q: Should I memorize solutions?**  
A: No! Understand patterns and approach. Memorized code shows in real interview.

**Q: How many problems should I solve?**  
A: At least 50-80 problems before interview. More is better.

**Q: Is system design harder than algorithms?**  
A: Different, not harder. Algorithms have correct answers. System design is design discussion.

**Q: What if I blank during interview?**  
A: Ask for hints, think out loud, propose brute force first. Interviewers don't expect perfection.
