# Pedagogic Guide - DDD

## Learning Path

### Phase 1: Fundamentals
1. Learn ubiquitous language concept
2. Understand entities vs value objects
3. Identify aggregates

### Phase 2: Intermediate
1. Design bounded contexts
2. Repository pattern implementation
3. Domain services identification

### Phase 3: Advanced
1. Strategic DDD (context mapping)
2. Event-driven DDD
3. DDD + Microservices

## Building Blocks

| Block | Description |
|-------|-------------|
| Entity | Object with identity |
| Value Object | Immutable, no identity |
| Aggregate | Cluster with root |
| Repository | Collection-like access |
| Domain Service | Business logic |

## Best Practices
- Start with domain language
- Focus on behavior, not just data
- Keep aggregates small
- Use factories for complex creation
