# Pedagogic Guide - Spring Modulith

## Learning Path

### Phase 1: Fundamentals
1. Understand modular monolith
2. Define application modules
3. Configure module structure

### Phase 2: Intermediate
1. Module dependencies
2. Event-driven communication
3. Internal package visibility

### Phase 3: Advanced
1. Module testing
2. Maven module structure
3. Migration to microservices

## Key Concepts

| Concept | Description |
|---------|-------------|
| ApplicationModule | Defines module boundary |
| Module Dependency | Explicit dependencies |
| Internal | Package visibility |
| ApplicationModuleListener | Event handling |

## Benefits
- Clear boundaries
- Migration path to microservices
- Build-time verification
- Maintainable codebase

## When to Use
- Start simple, scale later
- Team size moderate
- Clear domain boundaries
