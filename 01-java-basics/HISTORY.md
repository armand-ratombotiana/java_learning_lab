# History of Java Basics

## Timeline of Java Evolution

### 1995 - Java 1.0
- Initial release by Sun Microsystems
- "Write Once, Run Anywhere" vision
- Basic types: int, long, float, double, boolean, char, byte, short

### 1997 - Java 1.1
- Introduced inner classes
- JavaBeans specification

### 2000 - Java 1.3
- Improved performance
- HotSpot JVM introduced

### 2004 - Java 5 (1.5)
- **Major addition**: Enhanced for loop (for-each)
- Annotations introduced
- Autoboxing/unboxing
- Varargs (...)

### 2014 - Java 8
- Lambda expressions (affects basics usage)
- Method references
- Default methods in interfaces

### 2017 - Java 9
- Module system (JPMS)
- Private methods in interfaces

### 2021 - Java 17 (LTS)
- Sealed classes
- Pattern matching for switch
- Records

### 2023 - Java 21
- Virtual threads
- Pattern matching enhancements
- Sequenced collections

## Key Design Decisions

1. **Primitive types**: Chosen for performance (not objects)
2. **Arrays**: Zero-indexed (like C)
3. **Type safety**: Compile-time checking over runtime flexibility
4. **Garbage collection**: Automatic memory management from day one
