# Theory: Vector Database

## Vector Embeddings
High-dimensional vector representations of data (text, images, audio) where semantic similarity corresponds to vector proximity. Generated by embedding models (Sentence-BERT, CLIP, OpenAI Ada).

## Approximate Nearest Neighbor (ANN) Search
Brute-force kNN is O(n*d). ANN algorithms trade recall for speed. Common approaches:
- **HNSW** (Hierarchical Navigable Small World): Multi-layer graph navigation
- **IVF** (Inverted File Index): Partition vectors into clusters, search only nearest clusters
- **PQ** (Product Quantization): Compress vectors for memory efficiency

## HNSW Algorithm
Builds a multi-layer graph where upper layers have fewer connections (long-range edges), lower layers have dense connections. Search starts at top layer and greedily descends. Insertion: find nearest neighbors at each layer, connect to them.

## IVF Algorithm
K-means clusters the vector space. At search time, the query is compared to cluster centroids, and only the nearest w centroids' clusters are searched. Trade-off: nprobe (clusters to search) vs recall.

## Product Quantization
Split vector into sub-vectors, quantize each sub-vector independently using a codebook. Reduces memory by 10-100x with acceptable recall loss. Combined with IVF for efficient large-scale search.
