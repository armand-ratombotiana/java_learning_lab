# Mental Models: Fraud Detection

- **Fraud = Needle in a Haystack**: Legitimate transactions are the haystack; fraud is the needle. Need efficient filtering.
- **Rule Engine = Sieve**: Coarse filters catch bulk fraud; fine filters catch sophisticated attacks.
- **ML Model = Statistical Microscope**: Detects patterns invisible to rule engines (subtle correlations).
- **Feature = Behavioral Fingerprint**: Capture how a user typically behaves; deviations signal fraud.
- **Velocity = Acceleration Sensor**: Detects sudden bursts of activity that indicate automated attacks.
- **Score = Probability of Fraud**: Not a binary classifier; risk-based decisioning with thresholds.
