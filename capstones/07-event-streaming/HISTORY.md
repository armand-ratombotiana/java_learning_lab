# History: Event Streaming

- **2011**: LinkedIn open-sources Kafka — distributed commit log for event streaming
- **2013**: Kafka becomes Apache top-level project; adopted by Netflix, Uber, Twitter
- **2015**: Kafka Streams introduced — stream processing without external cluster
- **2016**: Exactly-once semantics (EOS) added
- **2018**: KSQL (now ksqlDB) — stream processing with SQL
- **2020s**: Kafka becomes ubiquitous; KRaft (remove ZooKeeper dependency) in 2.8+; 3.0+ fully KRaft
- **2024+**: Tiered storage (hot/warm/cold), shared storage architecture improvements
