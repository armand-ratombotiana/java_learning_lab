# History: Distributed Cache

- **1999**: Memcached created by LiveJournal — distributed memory object caching
- **2003**: Memcached widely adopted; simple key-value with consistent hashing
- **2006**: Voldemort (LinkedIn) — distributed key-value with replication
- **2009**: Redis released — in-memory data structure store with persistence
- **2011**: Redis Cluster (distributed mode) — consistent hashing, gossip, failover
- **2013**: Hazelcast — Java-based in-memory data grid
- **2015+**: Memcached vs Redis vs Hazelcast; distributed caching becomes standard infrastructure
- **2020s**: Tiered caching (memory + SSD), multi-region caching, global cache with async replication
