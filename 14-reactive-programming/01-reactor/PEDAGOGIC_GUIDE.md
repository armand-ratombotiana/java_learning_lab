# Pedagogic Guide: Project Reactor

## Learning Path

### Phase 1: Foundation (Day 1)
- Understand reactive programming
- Learn Flux and Mono
- Create first reactive streams

### Phase 2: Core Skills (Day 2-3)
- Master operators
- Handle errors reactively
- Manage backpressure

### Phase 3: Advanced (Day 4-5)
- Optimize reactive streams
- Use parallel processing
- Integrate with Spring

## Key Concepts

| Concept | Description |
|---------|-------------|
| Flux | Reactive stream of 0+ items |
| Mono | Reactive stream of 0/1 item |
| Operator | Transformation in stream |
| Backpressure | Consumer-driven flow control |

## Prerequisites
- Java 8+ knowledge
- Lambda/expressions

## Common Pitfalls
- Blocking in reactive code
- Forgetting to subscribe
- Memory leaks from unbounded streams
