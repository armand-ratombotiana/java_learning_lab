# Mini Project: The Generic Container & Type Token Resolver

## Objective
Build a system that demonstrates the limitations of type erasure and how to bypass them. You will intentionally trigger a Heap Pollution `ClassCastException`, observe a Bridge Method via reflection, and implement the "Super Type Token" pattern to extract generic metadata at runtime.

## Prerequisites
*   Java 17+

## Step 1: The Heap Pollution Trap
Demonstrate how mixing raw types with generics defeats compiler safety.

```java
import java.util.ArrayList;
import java.util.List;

public class HeapPollutionDemo {
    
    @SuppressWarnings({"unchecked", "rawtypes"}) // We suppress warnings to simulate legacy code
    public static void runDemo() {
        System.out.println("--- 1. Heap Pollution ---");
        
        List<String> stringList = new ArrayList<>();
        
        // 1. Assign to a raw type (Allowed for backward compatibility)
        List rawList = stringList; 
        
        // 2. Insert the wrong type! The compiler allows this because rawList is just a List of Objects.
        rawList.add(42); 
        
        System.out.println("Inserted an Integer into a List<String> successfully.");
        
        // 3. The Trap springs!
        try {
            // When we iterate, the compiler inserts an implicit cast to String: (String) item
            for (String item : stringList) { 
                System.out.println(item);
            }
        } catch (ClassCastException e) {
            System.err.println("CRASH! Heap Pollution detected: " + e.getMessage());
        }
    }
}
```

## Step 2: Observing Bridge Methods
Create a generic hierarchy and use reflection to prove the compiler generated a synthetic method.

```java
import java.lang.reflect.Method;

// The Generic Interface
interface Node<T> {
    void setData(T data);
}

// The Concrete Class
class StringNode implements Node<String> {
    @Override
    public void setData(String data) {
        System.out.println("Setting string: " + data);
    }
}

public class BridgeMethodDemo {
    public static void runDemo() {
        System.out.println("\n--- 2. Bridge Methods ---");
        
        Method[] methods = StringNode.class.getDeclaredMethods();
        for (Method m : methods) {
            if (m.getName().equals("setData")) {
                System.out.print("Method: " + m.getName() + "(" + m.getParameterTypes()[0].getSimpleName() + ")");
                if (m.isBridge()) {
                    System.out.println(" <-- SYNTHETIC BRIDGE METHOD (Generated by Compiler)");
                } else {
                    System.out.println(" <-- YOUR ACTUAL METHOD");
                }
            }
        }
    }
}
```

## Step 3: The Super Type Token (Bypassing Erasure)
Local generic variables are erased, but *class definitions* retain their generic metadata in the compiled bytecode. We can use an anonymous subclass to capture this data.

```java
import java.lang.reflect.ParameterizedType;
import java.lang.reflect.Type;
import java.util.List;

// Abstract base class forces subclassing
abstract class TypeReference<T> {
    private final Type type;

    protected TypeReference() {
        // 1. Get the generic superclass of the current instance
        Type superclass = getClass().getGenericSuperclass();
        
        if (superclass instanceof Class) {
            throw new RuntimeException("Missing type parameter.");
        }
        
        // 2. Extract the actual type argument (e.g., List<String>)
        ParameterizedType parameterized = (ParameterizedType) superclass;
        this.type = parameterized.getActualTypeArguments()[0];
    }

    public Type getType() {
        return type;
    }
}

public class TypeTokenDemo {
    public static void runDemo() {
        System.out.println("\n--- 3. Super Type Tokens ---");
        
        // The {} creates an anonymous subclass of TypeReference!
        // This embeds the generic type information into the .class file.
        TypeReference<List<String>> stringListRef = new TypeReference<List<String>>() {};
        TypeReference<List<Integer>> intListRef = new TypeReference<List<Integer>>() {};

        System.out.println("Captured Type 1: " + stringListRef.getType());
        System.out.println("Captured Type 2: " + intListRef.getType());
    }
}
```
