## 💻 Common Code Patterns

### Creating Objects
```java
// Traditional class
Person person = new Person("Alice", 25);

// Record (Java 21)
record Person(String name, int age) {}
Person person = new Person("Alice", 25);

// Builder pattern
Person person = new Person.Builder()
    .name("Alice")
    .age(25)
    .build();
```

### Processing Collections
```java
// Traditional loop
for (String item : items) {
    System.out.println(item);
}

// Streams
items.stream()
    .filter(item -> item.length() > 3)
    .map(String::toUpperCase)
    .forEach(System.out::println);
```

### Exception Handling
```java
// Try-catch
try {
    // code
} catch (IOException e) {
    // handle
} finally {
    // cleanup
}

// Try-with-resources
try (FileReader reader = new FileReader("file.txt")) {
    // code
} catch (IOException e) {
    // handle
}
```

### Concurrency
```java
// Thread creation
Thread thread = new Thread(() -> {
    // code
});
thread.start();

// ExecutorService
ExecutorService executor = Executors.newFixedThreadPool(10);
executor.submit(() -> {
    // code
});
executor.shutdown();
```

## 📈 Performance Considerations

### Collections
- **ArrayList:** Fast random access, slow insertion
- **LinkedList:** Slow random access, fast insertion
- **HashMap:** Fast lookup, unordered
- **TreeMap:** Ordered, slower than HashMap

### Streams
- **Sequential:** Single-threaded processing
- **Parallel:** Multi-threaded processing
- **Lazy Evaluation:** Operations deferred until terminal
- **Short-circuit:** Some operations stop early

### Concurrency
- **Synchronization:** Prevents race conditions, impacts performance
- **Locks:** More flexible than synchronized
- **Concurrent Collections:** Thread-safe without synchronization
- **Thread Pools:** Reuse threads, reduce overhead

### I/O
- **Buffered I/O:** Faster than unbuffered
- **NIO:** Better for large files
- **Serialization:** Slower than custom formats
- **Compression:** Reduces size, increases CPU
