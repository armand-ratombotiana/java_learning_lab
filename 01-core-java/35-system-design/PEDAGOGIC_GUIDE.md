# System Design - Learning Path

## Beginner
- Understand monolithic vs microservices
- Know horizontal vs vertical scaling
- Learn basic database normalization
- Understand RESTful API design

## Intermediate
- Design for scalability (caching, load balancing)
- Apply CAP theorem trade-offs
- Use circuit breaker and retry patterns
- Choose appropriate databases (SQL vs NoSQL)

## Advanced
- Design complex distributed systems
- Implement event-driven architecture
- Optimize for performance at scale
- Handle data consistency across services

## Key Concepts
- Architecture patterns (monolithic, microservices, layered)
- Scalability strategies (replication, sharding, partitioning)
- CAP theorem and consistency models
- Fault tolerance patterns

## Assessment
- Design a system for a given problem
- Explain trade-offs in design decisions
- Scale a design from 1K to 1M users
