# Pedagogic Guide: Chaos Engineering

## Learning Path
1. **Principles** (1h): Resilience, steady state
2. **Injection** (2h): Latency, errors, failures
3. **Tools** (2h): Chaos Monkey, Litmus
4. **Practice** (1h): GameDays, experiments

## Assessment
- [ ] Inject failures in service
- [ ] Implement resilience patterns
- [ ] Design chaos experiment
