# Clean Architecture - Learning Path

## Beginner
- Understand layer responsibilities
- Know the dependency rule
- Learn entity vs data transfer object
- Understand use case concept

## Intermediate
- Implement clean architecture in a new project
- Apply dependency injection properly
- Create proper abstraction boundaries
- Test use cases in isolation

## Advanced
- Handle cross-layer concerns (logging, security)
- Scale architecture for large systems
- Integrate with frameworks (Spring, Jakarta EE)
- Manage database transactions in use cases

## Key Concepts
- Domain Layer: entities, business rules
- Application Layer: use cases, input/output ports
- Infrastructure Layer: adapters, external services
- Presentation Layer: controllers, API handlers

## Framework Integration
- Spring: @Service, @Repository, @Controller
- CDI/Jakarta: @ApplicationScoped, @RequestScoped
- Custom: use case executors

## Assessment
- Design layer structure for a domain
- Implement use case with dependencies
- Explain dependency direction choices
