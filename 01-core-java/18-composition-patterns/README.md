# Module 18: Composition Patterns

## Overview
This module explores the fundamental Object-Oriented principle of "Favor Composition over Inheritance". It dives deep into structural design patterns that utilize object composition to achieve runtime flexibility, avoid class explosion, and build complex tree structures.

## Learning Objectives
By the end of this module, you will be able to:
1. Explain why composition is generally preferred over inheritance in modern software design.
2. Implement the Decorator pattern to add behavior to objects dynamically at runtime.
3. Implement the Composite pattern to treat individual objects and collections of objects uniformly.
4. Utilize Delegation and Wrapper patterns to forward method calls.
5. Identify the edge cases of composition, including the Object Identity problem and boilerplate overhead.
6. Compare and contrast Decorator, Proxy, and Composite patterns.
