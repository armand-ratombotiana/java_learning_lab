# Code Organization - Learning Path

## Beginner
- Understand package organization (layer vs feature)
- Learn Java naming conventions (camelCase, PascalCase)
- Know basic SOLID explanation
- Understand standard Maven/Gradle structure

## Intermediate
- Apply all 5 SOLID principles in code
- Refactor existing code to follow best practices
- Design package structure for new projects
- Use dependency injection properly

## Advanced
- Design package boundaries for large systems
- Create internal libraries with clean APIs
- Balance cohesion and coupling
- Implement hexagonal architecture basics

## Key Concepts
- Single Responsibility Principle
- Open/Closed Principle
- Liskov Substitution Principle
- Interface Segregation Principle
- Dependency Inversion Principle

## Assessment
- Identify SOLID violations in code
- Design proper package structure for a domain
- Refactor code to follow SOLID principles
