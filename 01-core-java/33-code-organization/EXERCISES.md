# Code Organization Exercises

## Exercise 1: Refactor Multi-Responsibility Class
Take a class with 3 responsibilities (e.g., UserManager with persistence, validation, email) and split into separate classes following SRP.

## Exercise 2: Interface Segregation
Given a fat interface with 10 methods, split into 3-4 focused interfaces for different use cases.

## Exercise 3: Fix Liskov Substitution
Given a broken inheritance hierarchy (e.g., Bird -> Penguin), fix using proper interface segregation.

## Exercise 4: Apply Dependency Inversion
Refactor a class that directly instantiates dependencies to use constructor injection.

## Exercise 5: Organize by Feature
Take 10 scattered classes and organize them into proper feature-based packages with clear boundaries.
