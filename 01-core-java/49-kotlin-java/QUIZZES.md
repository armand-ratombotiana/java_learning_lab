## Q1: Null Safety
What does the Elvis operator `?:` do in Kotlin?

A) It forces a nullable variable to throw a NullPointerException if it is null.
B) It executes a block of code asynchronously.
C) It returns the left-hand operand if it is not null; otherwise, it returns the right-hand operand (default value).
D) It compares two objects for deep equality.

**Answer**: C
**Explanation**: The Elvis operator (`val x = name ?: "Unknown"`) provides a concise way to handle null fallbacks, replacing verbose `if (name != null) name else "Unknown"` checks in Java.

## Q2: Data Classes
Which of the following methods are automatically generated by the Kotlin compiler for a `data class`?

A) `equals()`, `hashCode()`, `toString()`, and `copy()`
B) `start()`, `stop()`, and `run()`
C) `saveToDatabase()` and `delete()`
D) Only `toString()`

**Answer**: A
**Explanation**: `data class` is designed specifically to hold data. The compiler automatically derives `equals()`, `hashCode()`, `toString()`, `copy()`, and `componentN()` functions directly from the properties declared in the primary constructor, drastically reducing boilerplate.

## Q3: Java Interop
When you define a top-level function `fun calculate()` in a file named `Utils.kt`, how do you call this function from a Java class?

A) `new Utils().calculate();`
B) `UtilsKt.calculate();`
C) `Kotlin.calculate();`
D) It is impossible to call top-level Kotlin functions from Java.

**Answer**: B
**Explanation**: The Kotlin compiler creates a facade Java class named after the file (with `Kt` appended) and translates the top-level functions into static methods within that class.
