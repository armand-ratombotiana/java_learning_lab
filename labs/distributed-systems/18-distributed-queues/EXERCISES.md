# Exercises — Distributed Queues

1. **In-Memory Queue (Easy)**: Implement basic producer-consumer queue
2. **Partitioned Queue (Medium)**: Add partition logic with routing
3. **Message Dedup (Medium)**: Implement deduplication with time window
4. **Dead Letter Queue (Easy)**: Add DLQ with max retry policy
5. **Exponential Backoff (Medium)**: Implement retry with backoff
6. **Pulsar Producer (Medium)**: Implement Pulsar message producer
7. **Pulsar Consumer (Medium)**: Implement Pulsar consumer with ack
8. **Exactly-Once (Hard)**: Implement end-to-end exactly-once semantics
9. **Queue Monitoring (Easy)**: Add metrics for queue depth and latency
10. **Replay Consumer (Hard)**: Implement consumer with position replay
