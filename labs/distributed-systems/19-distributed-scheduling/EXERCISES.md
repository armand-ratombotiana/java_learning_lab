# Exercises — Distributed Scheduling

1. **Cron Parser (Easy)**: Implement 5-field cron parser
2. **Next Fire Time (Medium)**: Calculate next fire time from expression
3. **Job Store (Medium)**: Implement persistent job storage
4. **Leader Election (Medium)**: Implement ZK-based leader for job execution
5. **Partitioned Scheduler (Hard)**: Implement hash-based job distribution
6. **Misfire Handler (Medium)**: Implement fire-now and ignore policies
7. **Quartz Cluster (Hard)**: Configure and run Quartz cluster with JDBC store
8. **Recovery (Hard)**: Implement job recovery after scheduler crash
9. **Dynamic Scheduling (Easy)**: Add/remove jobs at runtime
10. **Cron Calendar (Medium)**: Add exclusion calendar support (holidays)
