# History of CI/CD

- **1991**: Continuous Integration concept coined by Grady Booch (in object-oriented design context).
- **1997-1999**: Extreme Programming (XP) popularizes CI as a practice.
- **2000**: CruiseControl — first open-source CI server (Java).
- **2005**: Hudson CI server released (predecessor to Jenkins).
- **2007**: ThoughtWorks Go (later GoCD) released.
- **2011**: Hudson → Jenkins fork after Oracle acquisition of Sun.
- **2014**: Travis CI popularizes SaaS CI for open-source; CircleCI launches.
- **2015**: Docker integration transforms CI/CD pipelines.
- **2016**: GitLab CI/CD introduced; Serverless CI/CD concepts emerge.
- **2018**: GitHub Actions announced.
- **2019-2024**: GitHub Actions GA; GitOps (ArgoCD, Flux) gains traction; AI-assisted pipelines.

The evolution from manual releases to fully automated GitOps represents a fundamental shift in software engineering.
