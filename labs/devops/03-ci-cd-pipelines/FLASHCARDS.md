# CI/CD Flashcards

**Q: What is Continuous Integration?**
A: Automatically building and testing every code commit.

**Q: What is Continuous Delivery?**
A: Automated deployment to staging; manual approval for production.

**Q: What is Continuous Deployment?**
A: Fully automated deployment to production after all tests pass.

**Q: What is a GitHub Actions workflow?**
A: A YAML file defining automated CI/CD jobs triggered by events.

**Q: What is a Jenkins Pipeline?**
A: A series of stages defined in a Jenkinsfile (Declarative or Scripted).

**Q: What is an artifact in CI/CD?**
A: The build output (Docker image, JAR, binary) stored for deployment.

**Q: What is a matrix build?**
A: Running the same job across multiple OS/version combinations in parallel.

**Q: What is "shift left" in CI/CD?**
A: Moving testing and validation earlier in the development lifecycle.

**Q: What does "build once, deploy many" mean?**
A: Build a single artifact and promote it through all environments unchanged.

**Q: What is a webhook?**
A: HTTP callback triggered by repository events (push, PR) to start CI/CD.
