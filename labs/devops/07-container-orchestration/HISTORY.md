# History of Container Orchestration

- **2014**: Kubernetes v1.0 — basic replication controller (predecessor to deployments).
- **2015**: Kubernetes v1.2 — Deployments GA (replace ReplicationController).
- **2016**: Kubernetes v1.3 — Horizontal Pod Autoscaling (CPU-only), rolling updates.
- **2017**: Kubernetes v1.8 — HPA with custom metrics; StatefulSets GA.
- **2018**: Kubernetes v1.10 — Pod Priority and Preemption; HPA v2 (custom metrics).
- **2019**: Kubernetes v1.14 — Pod Priority GA; Windows containers GA.
- **2020**: Kubernetes v1.19 — Ingress v1; HPA v2beta2; Event-driven autoscaling (KEDA).
- **2021**: Kubernetes v1.21 — CronJobs GA; PodSecurityPolicy deprecated.
- **2022**: Kubernetes v1.24 — HPA with container-level metrics (alpha); VPA GA.
- **2023-2024**: KEDA 2.0+ (event-driven scaling); Cluster Autoscaler improvements; HPA with predictive scaling.

Orchestration evolved from basic replication to sophisticated, metric-driven, event-triggered scaling and deployment strategies.
