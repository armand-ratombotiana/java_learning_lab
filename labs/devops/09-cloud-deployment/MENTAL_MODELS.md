# Mental Models for Cloud Deployment

## 1. Utility Analogy
Cloud is like electricity: you don't build your own power plant — you plug into the grid and pay for what you use. Scale up when you need more, scale down when you don't.

## 2. LEGO Bricks
Cloud services are like LEGO bricks: combine managed services (RDS, SQS, S3, Lambda) to build applications. Choose the right brick for each function rather than building everything yourself.

## 3. Cloud as Operating System
Cloud is the "operating system" for modern applications:
- **Compute** = CPU
- **Storage** = Hard drive
- **Database** = RAM
- **Networking** = Motherboard bus
- **IAM** = User accounts
- **Monitoring** = Activity monitor

## 4. Gardening (Managed Services)
Running your own database = gardening (plant, water, weed, harvest). Managed databases = buying vegetables (ready to use, delivered fresh). Choose managed services when possible.
