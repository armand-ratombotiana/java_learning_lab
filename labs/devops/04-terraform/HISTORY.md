# History of Terraform

- **2014 (Jul)**: Terraform v0.1 released by HashiCorp (creators of Vagrant, Packer, Consul).
- **2015**: Open source release; core concepts (providers, resources, state) established.
- **2016**: Terraform v0.7 — state locking, remote state backends (S3, Consul).
- **2017**: Modules registry launched; `terraform validate` introduced.
- **2018**: Terraform v0.12 — major HCL improvements (first-class expressions, `for_each`).
- **2019**: Terraform Cloud launched; Terraform v0.13 — provider installation improvements.
- **2020**: HashiCorp goes public; Terraform v0.14 — improved error messages, provider mirrors.
- **2021**: Terraform v1.0 — stable release, backward compatibility promise.
- **2022**: BSL license change; OpenTofu fork created by Linux Foundation.
- **2023-2024**: Terraform v1.5 — config-driven import, check blocks; v1.6 — test framework; v1.7 — remove blocks, S3 backend native state locking.

Terraform defined modern IaC practices and inspired many tools in the ecosystem.
