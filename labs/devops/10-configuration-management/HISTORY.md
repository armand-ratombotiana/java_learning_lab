# History of Configuration Management

- **1990s**: Shell scripts, cfengine (Mark Burgess, 1993) — early automation.
- **2005**: Puppet announced by Luke Kanies (Reductive Labs). Ruby-based DSL.
- **2008**: Opscode launches Chef (Adam Jacob). Ruby DSL; hosted Chef platform.
- **2012**: Ansible announced by Michael DeHaan (also created Cobbler, Func). Agentless SSH.
- **2013**: Ansible acquired by Red Hat; SaltStack launched.
- **2014**: Ansible Tower (enterprise GUI) released.
- **2015**: Ansible becomes most popular; Chef and Puppet add Windows support.
- **2017**: Red Hat Ansible Automation Platform; Puppet acquires Cloudsmith.
- **2018-2020**: Immutable infrastructure trend reduces CM importance; containers replace config management for app deployment.
- **2021-2024**: Ansible continues dominance for server provisioning; infrastructure as code (Terraform) complements CM.

CM tools evolved from scripting to mature platforms, then adapted to the container/Kubernetes era.
