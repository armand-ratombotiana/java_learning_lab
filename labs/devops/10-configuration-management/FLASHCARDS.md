# Configuration Management Flashcards

**Q: What is Ansible?**
A: Agentless configuration management tool using SSH (push model).

**Q: What is idempotency?**
A: Running same operation multiple times produces same result.

**Q: What is a playbook?**
A: YAML file defining configuration policies (hosts, tasks, variables).

**Q: What is a role in Ansible?**
A: Organizational unit for tasks, handlers, templates, and variables.

**Q: What is Ansible Vault?**
A: Tool for encrypting sensitive data (passwords, keys) in Ansible.

**Q: What is a handler?**
A: Task that runs on notify at end of play (e.g., restart service).

**Q: What is a manifest in Puppet?**
A: .pp file with resource declarations (desired state).

**Q: What is a cookbook in Chef?**
A: Collection of recipes, templates, and files for configuration.

**Q: What is configuration drift?**
A: Server state diverging from intended configuration over time.

**Q: What is Ansible Galaxy?**
A: Community hub for sharing Ansible roles.
