# 08-Service Mesh - Istio & Service Mesh

## Overview
A service mesh provides a dedicated infrastructure layer for service-to-service communication. This lab covers Istio, sidecar proxy, traffic management, mTLS, and observability.

## Prerequisites
- Kubernetes fundamentals (Lab 02)
- Advanced orchestration (Lab 07)

## Learning Objectives
- Understand service mesh architecture and sidecar proxy pattern
- Deploy and configure Istio service mesh
- Implement traffic management (routing, mirroring, fault injection)
- Enable mTLS for service-to-service encryption
- Use service mesh for observability (metrics, traces, access logs)
