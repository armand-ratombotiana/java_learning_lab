# History of Secrets Management

- **1970s-1990s**: Passwords stored in plain text files, databases, or hardcoded.
- **2000s**: SSH keys for automation; password managers for humans (1Password, LastPass).
- **2010**: Mozilla announced (open-source secrets management for cloud).
- **2012**: Keywhiz released by Square (Java-based secrets management).
- **2014**: HashiCorp Vault v0.1 announced (first major dedicated secrets management tool).
- **2015**: Vault 0.2 — dynamic secrets, transit encryption, auditing.
- **2016**: Vault 0.6 — AWS IAM auth, Google Cloud auth, Kubernetes integration.
- **2017**: Vault 0.9 — database secrets engine (auto-rotation); Vault joins CNCF.
- **2018-2019**: Vault 1.0 — stability and production features; Vault Enterprise.
- **2020-2021**: Vault 1.5 — integrated storage (Raft); Vault Agent caching.
- **2022-2024**: Vault 1.12+ — namespaces, ACME/PKI improvements, OIDC enhancements; cloud provider native secrets (AWS Secrets Manager, GCP Secret Manager, Azure Key Vault).

The field evolved from ad-hoc file storage to purpose-built, highly secure secrets platforms.
