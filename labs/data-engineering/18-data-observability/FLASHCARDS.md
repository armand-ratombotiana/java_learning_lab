# Flashcards: Data Observability

## Card 1
**Front**: What are the 5 pillars of data observability?
**Back**: Freshness (recency), Distribution (statistical), Volume (completeness), Schema (structure), Lineage (dependencies)

## Card 2
**Front**: What is Soda?
**Back**: Open-source data observability tool with SQL-based checks for freshness, volume, schema, and custom rules

## Card 3
**Front**: What is anomaly detection in data?
**Back**: Identifying patterns: volume spikes, freshness delays, distribution shifts, schema changes that deviate from historical norms

## Card 4
**Front**: What is data profiling?
**Back**: Automated statistical analysis: row counts, null rates, distinct counts, min/max, distribution shapes, patterns

## Card 5
**Front**: What is data incident management?
**Back**: Process: detect anomaly -> classify severity -> notify owner -> investigate -> resolve -> prevent recurrence
