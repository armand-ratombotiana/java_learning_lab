# Exercises for Data Observability

1. Set up Soda Core and configure data source connections
2. Write Soda checks for freshness, volume, schema, and custom SQL
3. Implement anomaly detection for row count and distribution changes
4. Build data profiling pipeline (histograms, null rates, distinct counts)
5. Create observability dashboards with trend charts and alerts
6. Design incident response workflow for data quality incidents
