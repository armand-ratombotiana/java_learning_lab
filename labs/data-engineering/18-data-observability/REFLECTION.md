# Reflection: Data Observability

- Observability shifts data quality from reactive (fix after failure) to proactive (detect before impact)
- Anomaly detection is most valuable for unknown unknowns — issues you didn't think to check
- Data lineage is essential for both root cause analysis and impact assessment
- Start with the 5 pillars and expand coverage iteratively
- Alert fatigue is a real risk; tune thresholds and severity levels carefully
