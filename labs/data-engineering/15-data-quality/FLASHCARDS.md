# Flashcards: Data Quality Engineering

## Card 1
**Front**: What are the six data quality dimensions?
**Back**: Accuracy, Completeness, Consistency, Timeliness, Validity, Uniqueness

## Card 2
**Front**: What is a Great Expectations Expectation Suite?
**Back**: Collection of expectations (quality rules) applied to a dataset for validation

## Card 3
**Front**: What is a data contract?
**Back**: Formal agreement specifying schema, quality SLAs, ownership, and change procedures between producers and consumers

## Card 4
**Front**: What does schema drift mean?
**Back**: Unexpected changes to data schema (new columns, type changes, missing columns) that can break pipelines

## Card 5
**Front**: What is a dbt test?
**Back**: SQL query that returns failing rows; built-in: unique, not_null, accepted_values, relationships
