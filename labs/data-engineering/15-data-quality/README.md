# Data Quality Engineering

## Overview
Data quality engineering ensures that data meets standards for accuracy, completeness, consistency, timeliness, and validity. This lab covers Great Expectations, dbt tests, data contracts, SLAs, schema drift detection, and automated quality monitoring pipelines.

## Key Concepts
- **Quality Dimensions**: Accuracy, completeness, consistency, timeliness, validity, uniqueness
- **Great Expectations**: Open-source data quality framework with automated expectation suites
- **dbt Tests**: Built-in and custom data tests in dbt transformations
- **Data Contracts**: Formal agreements between data producers and consumers
- **Schema Drift**: Detection and handling of unexpected schema changes
- **Quality SLAs**: Service Level Agreements for data freshness, volume, and quality metrics

## Learning Objectives
1. Define and measure the six dimensions of data quality
2. Implement automated quality checks with Great Expectations
3. Write dbt tests for data quality validation
4. Design data contracts between producers and consumers
5. Detect and handle schema drift in production pipelines
6. Set up monitoring dashboards for quality SLAs
