# Exercises for Data Quality Engineering

1. Define expectation suite for a dataset (null rates, value ranges, uniqueness)
2. Implement Great Expectations checkpoint with Slack alerting
3. Write dbt tests: unique, not_null, accepted_values, relationships, custom
4. Create a data contract document with schema, SLAs, and ownership
5. Build schema drift detection: compare incoming vs expected schema
6. Create quality dashboard with trend metrics over time
