# Data Quality Theory

## Six Dimensions
1. **Accuracy**: Data reflects real-world values
2. **Completeness**: All required data present
3. **Consistency**: Consistent across systems
4. **Timeliness**: Current and available
5. **Validity**: Conforms to formats/rules
6. **Uniqueness**: No duplicates

## Check Types
- Freshness: How recent?
- Volume: Expected counts
- Schema: Column types/constraints
- Content: Value ranges, patterns
- Referential: FK relationships
