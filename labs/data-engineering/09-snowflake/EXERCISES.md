# Exercises for Snowflake Data Cloud

1. Create warehouses of sizes X-Small through Large; run queries on each and compare latency and credit usage
2. Set up Snowpipe for auto-ingestion from cloud storage; monitor pipe status and error handling
3. Analyze clustering depth on a large table; design optimal clustering keys based on query patterns
4. Simulate accidental data deletion; recover using Time Travel AT/BEFORE; verify data integrity
5. Clone a production database for testing; run ETL transformations; compare storage costs before and after
6. Create a share, add reader accounts, grant permissions, and verify cross-account data access
