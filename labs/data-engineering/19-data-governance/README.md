# Data Governance

## Overview
Data governance defines policies, processes, and technologies for managing data assets. This lab covers RBAC for data, PII detection and masking, data classification, GDPR/compliance frameworks, data retention, and audit logging for enterprise data platforms.

## Key Concepts
- **Data Governance**: Framework of policies, processes, and controls for data management
- **RBAC**: Role-Based Access Control for granular data permissions
- **PII Detection**: Automated identification of personally identifiable information
- **Data Masking**: Protecting sensitive data through obfuscation techniques
- **GDPR/Compliance**: Regulatory frameworks for data privacy and protection
- **Data Retention**: Lifecycle policies for data storage and deletion

## Learning Objectives
1. Design and implement RBAC for data platform access control
2. Build automated PII detection using patterns and classifiers
3. Implement data masking strategies (static, dynamic, on-the-fly)
4. Apply GDPR compliance requirements to data pipelines
5. Establish data retention and deletion policies
6. Set up audit logging for data access and governance operations
