# Mental Models for Data Governance

## 1. The Constitution
Data governance is like a constitution: sets the rules (policies), defines branches (roles), establishes rights (data privacy), and provides enforcement mechanisms (audit, penalties).

## 2. The City Planning
Data platform is a city. Governance = zoning laws. RBAC = building access. Masking = window blinds. PII detection = police identifying unsafe buildings. Audit = security cameras.

## 3. The Bank Vault
Data governance is bank security. Vault = encrypted data. Access cards = RBAC. Security cameras = audit logs. Bank manager = data owner. Customers = data subjects with rights.
