# Messaging - HISTORY

## Timeline

### 1970s-1980s: Early Message Queues
- IBM RSCS (1980) — virtual machine communication
- DEC message system (1984) — first email-based messaging

### 1990s: Enterprise Messaging
- 1992: IBM MQ Series — first commercial message queue
- 1994: TIBCO Rendezvous — pub/sub messaging
- 1998: Java Message Service (JMS) specification
- 1999: ActiveMQ project starts (later becomes Apache ActiveMQ)

### 2000s: Open Source and AMQP
- 2003: AMQP (Advanced Message Queuing Protocol) specification starts
- 2006: RabbitMQ founded (AMQP implementation)
- 2007: Apache ActiveMQ reaches maturity
- Stream processing emerges (STREAM project at Stanford)

### 2010s: Stream Processing Revolution
- 2011: LinkedIn open-sources Apache Kafka
- 2012: Kafka adopted by Netflix, Uber, Spotify
- 2013: Apache Storm for real-time stream processing
- 2015: Apache Flink (stream processing with event time)
- 2016: Apache Kafka Streams API (stream processing in Java)
- 2018: Confluent Cloud (Kafka as a service)

### 2020s: Cloud-Native, Kinesis, Pulsar
- Apache Pulsar gains traction (multi-tenant, geo-replication)
- Amazon MSK (Managed Streaming for Kafka)
- Serverless messaging (AWS SQS, Google Pub/Sub)
- Event-driven microservices with Spring Cloud Stream

## Java Messaging Evolution
- JMS (1998) — standard API for message queues
- Spring JMS (2004) — simplified JMS template
- Spring Kafka (2015) — Kafka integration
- Spring Cloud Stream (2016) — event-driven microservices
- Spring Cloud Function + Stream (2019) — functional event processing
