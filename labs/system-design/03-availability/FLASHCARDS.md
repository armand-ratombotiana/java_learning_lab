# Availability - FLASHCARDS

## Key Terms (30 cards)

1. **Availability**: Percentage of time system is operational
2. **SLA**: Service Level Agreement - contract with customers
3. **SLO**: Service Level Objective - internal target
4. **SLI**: Service Level Indicator - actual measurement
5. **RTO**: Recovery Time Objective - max restore time
6. **RPO**: Recovery Point Objective - max data loss time
7. **Redundancy**: Duplication of components for reliability
8. **Failover**: Automatic switching to backup
9. **Circuit Breaker**: Pattern to prevent cascade failures
10. **Health Check**: Mechanism to detect service failures
11. **Active-Active**: Multiple active instances
12. **Active-Passive**: One active, one standby
13. **N+1**: N instances plus one standby
14. **Warm Standby**: Partially running backup
15. **Cold Standby**: Off-site backup, slower recovery
16. **Pilot Light**: Minimal running backup
17. **Multi-Region**: Deployment across geographic regions
18. **Graceful Degradation**: Reduced functionality on failure
19. **Idempotency**: Safe to retry operations
20. **Four Golden Signals**: Latency, Traffic, Errors, Saturation
21. **Mean Time Between Failures**: Average failure interval
22. **Mean Time to Recovery**: Average recovery time
23. **Chaos Engineering**: Testing system resilience
24. **Runbook**: Step-by-step recovery procedures
25. **Heartbeat**: Regular health signal
26. **Splinter Zone**: Isolated failure area
27. **Rolling Update**: Incremental deployment
28. **Blue-Green Deployment**: Parallel environments
29. **Canary Release**: Gradual rollout
30. **Disaster Recovery**: Procedures to recover from major failures
