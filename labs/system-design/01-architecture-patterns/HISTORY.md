# Architecture Patterns - HISTORY

## Timeline

### 1960s-1970s: Monolithic Era
- Mainframe programs with no separation of concerns
- All logic in single executable

### 1980s: Client-Server Emerges
- Separation of UI (client) from data (server)
- Two-tier architecture becomes standard

### 1990s: Layered Architecture
- Three-tier architecture formalized
- Java Enterprise (J2EE) popularizes MVC
- 1994: Gang of Four publishes Design Patterns

### 2000s: Service-Oriented Architecture
- SOA introduces service boundaries
- Enterprise Service Bus (ESB) pattern emerges
- XML/SOAP web services

### 2010s: Microservices Revolution
- 2011: First microservices workshop near Venice
- 2014: Martin Fowler's "Microservices" article
- 2015: Spring Cloud / Netflix OSS stack
- Docker and containerization accelerate adoption

### 2020s: Event-Driven & CQRS
- Kafka becomes de facto event streaming platform
- CQRS paired with Event Sourcing gains traction
- Serverless and FaaS challenge traditional patterns
- Service mesh (Istio, Linkerd) handles inter-service concerns

## Key Milestones in Java
- 1999: J2EE with EJB (Enterprise JavaBeans) — layered pattern
- 2004: Spring Framework 1.0 — inversion of control
- 2014: Spring Boot 1.0 — microservices made easy
- 2018: Spring Cloud Gateway — API gateway pattern
- 2020: Quarkus, Micronaut — cloud-native Java frameworks
