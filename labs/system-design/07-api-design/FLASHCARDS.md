# API Design - FLASHCARDS

### Card 1
**Q:** REST resource naming?
**A:** Use nouns, plural: /users not /getUser

### Card 2
**Q:** 201 status code?
**A:** Created - new resource successfully created.

### Card 3
**Q:** 401 vs 403?
**A:** 401 = not authenticated, 403 = authenticated but not authorized.

### Card 4
**Q:** URL versioning?
**A:** /api/v1/users, /api/v2/users

### Card 5
**Q:** Pagination best practice?
**A:** Cursor-based for large datasets, offset for small.

### Card 6
**Q:** OpenAPI?
**A:** API specification standard (formerly Swagger).
