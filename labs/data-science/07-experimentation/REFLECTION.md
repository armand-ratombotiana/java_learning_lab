# Reflection: Experimentation

## Key Takeaways

- A/B testing is the most reliable method for causal inference in product development
- Statistical significance ≠ practical significance — always report effect sizes
- Proper experiment design (power analysis, pre-registration, no peeking) prevents most errors
- Multiple testing correction is non-negotiable when evaluating many metrics
- Experiment culture is more important than experiment technology — ship null results, learn from failures

## Questions to Internalize

1. Would I bet my bonus on this experiment's result replicating?
2. Did I decide to stop this experiment based on the data, or based on my desire for a specific outcome?
3. What would happen if I ran this same experiment 50 times — how often would I get a false positive?
4. Is the metric I'm tracking truly aligned with the business outcome I care about?

## Growth Areas

| Skill | Beginner | Proficient | Master |
|---|---|---|---|
| Test design | A/B two groups | Factorial experiments | Switchback, MAB |
| Analysis | p-value, CI | CUPED, sequential | Bayesian + Thompson Sampling |
| Multiple testing | None | Bonferroni | FDR (BH procedure) |
| Metrics | Simple rate | Ratio metrics, CUPED | Surrogate metrics, OEC |
| Culture | Run experiments | Ship null results | Meta-experiments |
