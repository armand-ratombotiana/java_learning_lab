# Methods — Flashcards

1. **Q: Method signature?** A: Name + parameter types
2. **Q: Can return type distinguish overload?** A: No
3. **Q: Varargs must be?** A: Last parameter (only one)
4. **Q: Java passing mechanism?** A: Pass-by-value (always)
5. **Q: Static method can access?** A: Only static members
6. **Q: Recursion base case?** A: Condition that stops recursion
7. **Q: StackOverflowError cause?** A: Too many recursive calls
8. **Q: @Override annotation?** A: Compiler-checked override declaration
9. **Q: Can constructor be private?** A: Yes (singleton/factory)
10. **Q: Default constructor?** A: No-arg constructor from compiler
11. **Q: void return type?** A: Method returns nothing
12. **Q: Method inlining?** A: JIT replaces call with method body
13. **Q: Access modifier order?** A: public/private, then static/final, then return type
14. **Q: Main method signature?** A: public static void main(String[])
15. **Q: Can a method return multiple values?** A: No (use object/record)
16. **Q: `this` in static methods?** A: Not available
17. **Q: Method overloading?** A: Same name, different parameters
18. **Q: Varargs compile to?** A: Array
19. **Q: Assert keyword?** A: Debug-only check (enable with -ea)
20. **Q: Method area (Metaspace)?** A: Stores class bytecode and metadata
