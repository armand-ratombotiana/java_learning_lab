# Abstraction & Interfaces — Flashcards

1. **Q: Abstract class keyword?** A: abstract
2. **Q: Can abstract class have constructor?** A: Yes
3. **Q: Can abstract class have fields?** A: Yes
4. **Q: Interface keyword?** A: interface
5. **Q: implements keyword?** A: Class implements interface
6. **Q: Interface fields are?** A: public static final
7. **Q: Default method (Java 8)?** A: Interface method with body
8. **Q: Static method in interface (Java 8)?** A: Utility method in interface
9. **Q: Private method in interface (Java 9)?** A: Helper for default methods
10. **Q: @FunctionalInterface?** A: Exactly one abstract method
11. **Q: Multiple inheritance of type?** A: Class implements multiple interfaces
12. **Q: Diamond problem?** A: Conflicting default methods; must override
13. **Q: Abstract class vs interface?** A: Abstract = shared state; Interface = contract
14. **Q: Sealed interface (Java 17+)?** A: Controls which classes implement
15. **Q: Marker interface?** A: No methods (e.g., Serializable)
16. **Q: Lambda implements what?** A: Functional interface
17. **Q: Adapter pattern?** A: Wrap one interface with another
18. **Q: Template Method pattern?** A: Abstract class defines algorithm skeleton
19. **Q: Dependency Inversion?** A: Depend on abstractions, not concretions
20. **Q: Comparator functional interface?** A: int compare(T o1, T o2)
