# Reactive Testing -- Flashcards
Card 1: Q: Publisher? A: Emits data to Subscriber
Card 2: Q: Subscriber? A: Receives data from Publisher
Card 3: Q: Subscription? A: Manages demand (backpressure)
Card 4: Q: Flux? A: 0 to N items
Card 5: Q: Mono? A: 0 to 1 items
Card 6: Q: flatMap? A: Maps item to Publisher, flattens results
Card 7: Q: Scheduler? A: Thread pool for async execution
Card 8: Q: StepVerifier? A: Tests reactive streams
