## Real Production Scenarios
- **Twitter**: A base class in the notification system was modified for one use case, which broke all three downstream services
- **LinkedIn**: A `BaseDecorator` used for 12 subclasses; change in constructor signature required coordination across 8 different repos
- **Reddit**: A mixin pattern using interface default methods with read and write conflict; wrong resolution was causing exceptions

## Interview Patterns & Tips
- **Fragile base class problem**: A seemingly innocuous change to a base class can break derived classes in unexpected ways
- **Constructor ordering**: In a class hierarchy, the constructors are called in order from most base to most derived
- **Polymorphism and fields**: Fields don't have virtual behavior; if a subclass defines a field with the same name as a parent's field, it hides it
- **Member variables visibility**: Implement interfaces and use abstract methods to force extenders to define critical behavior
- **Override annotation**: Always use `@Override` when overriding a method to catch errors

## Deep Dive Questions
- **JVM**: How does the Vtable (virtual method table) work in the JVM? What is the index arrangement for inherited methods? How does a new class with additional methods affect the vtable?
- **Bytecode**: What is a bridge method? Give an example of covariant return and how the bridge method delegates to the actual overridden method.
- **JIT**: How does the JIT compiler handle virtual method dispatch in the presence of multiple derived types? What is a polymorphic inline cache (PIC) and how does it help monomorphic call site optimization?
- **Memory**: When a class is loaded, what structures are allocated in the metaspace? How are the class metadata related to instance creation via `new`?
- **Testing**: How can deep inheritance hierarchies cause difficulty in writing unit tests? Why is it difficult to mock subclass behavior?
