# OOP Basics — Reflection

1. How has OOP changed the way you design software compared to procedural programming?
2. Do you prefer `public` fields with direct access or getters/setters? Why?
3. How often do you use `this` explicitly vs relying on default behavior?
4. When should a class be immutable? How do you ensure immutability?
5. How do you decide what should be static vs instance members?
6. Have you used the builder pattern for complex object construction?
7. What's your experience with records? Do they replace classes for data carriers?
8. How do you design classes that are easy to test?
9. What does "programming to an interface, not an implementation" mean in practice?
10. How do you handle cross-cutting concerns (logging, security) in OOP design?
11. When is it appropriate to use inheritance vs composition?
12. What's the most reusable class you've written? What made it reusable?
