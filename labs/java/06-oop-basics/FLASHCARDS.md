# OOP Basics — Flashcards

1. **Q: What is a class?** A: Blueprint for objects
2. **Q: What is an object?** A: Instance of a class
3. **Q: Constructor name?** A: Same as class name
4. **Q: Constructor return type?** A: None (not even void)
5. **Q: this keyword?** A: Reference to current object
6. **Q: static field?** A: Shared across all instances (class variable)
7. **Q: Access modifier visibility order?** A: public > protected > default > private
8. **Q: Encapsulation?** A: Bundle data+methods, hide internals
9. **Q: Instance initializer block?** A: Runs before constructor body
10. **Q: Static initializer block?** A: Runs once when class loaded
11. **Q: Garbage collection?** A: Automatic memory reclamation
12. **Q: final field?** A: Cannot be reassigned after initialization
13. **Q: transient keyword?** A: Field not serialized
14. **Q: volatile keyword?** A: Field read/written from main memory
15. **Q: Package naming convention?** A: reverse domain: com.example.app
16. **Q: Record (Java 16+)?** A: Compact immutable data carrier
17. **Q: Fluent API?** A: Methods return this for chaining
18. **Q: Default constructor provided when?** A: When no constructor defined
19. **Q: Constructor delegation?** A: this() calls another constructor
20. **Q: Object lifecycle?** A: Declaration → new → initialization → usage → GC
