# Polymorphism — Quiz

1. What is polymorphism? **Many forms; same method call, different behavior**
2. What is dynamic method dispatch? **Method call resolved at runtime by object type**
3. What is overloading? **Multiple methods with same name, different params (compile-time)**
4. What is overriding? **Subclass provides specific implementation (runtime)**
5. Which is resolved at runtime? **Overriding**
6. What is a covariant return type? **Override method returns subtype (Java 5+)**
7. Can you overload a method by changing return type only? **No**
8. What is the vtable? **Virtual method table for dispatch**
9. What does instanceof do? **Checks if object is instance of a type**
10. What is the Open/Closed Principle? **Open for extension, closed for modification**
11. What is the Liskov Substitution Principle? **Subtype should work wherever supertype is expected**
12. Can static methods be polymorphic? **No (hidden, not overridden)**
13. What is itable? **Interface method table**
14. What is a megamorphic call site? **3+ types seen at same call site**
15. What is pattern matching for instanceof? **Type check + variable binding (Java 16+)**
16. What is the Strategy pattern? **Encapsulate algorithms, interchange at runtime**
17. What is the Command pattern? **Encapsulate requests as objects**
18. What does programming to an interface mean? **Using interface references instead of concrete types**
19. Can you use @Override on interface method implementations? **Yes**
20. What is the difference between overloading and overriding? **Overload=compile-time, same name/different params; Override=runtime, same signature**
