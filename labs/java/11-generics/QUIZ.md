# Generics — Self-Assessment Quiz

## Question 1

What is the primary purpose of generics in Java?

A) To improve runtime performance
B) To provide compile-time type safety
C) To enable reflection
D) To simplify garbage collection

<details>
<summary>Answer</summary>
B — Generics provide compile-time type safety by allowing types to be parameterized.
</details>

## Question 2

What does PECS stand for?

A) Parameterized Extends, Covariant Super
B) Producer Extends, Consumer Super
C) Primitive Extends, Class Super
D) Protected Extends, Concrete Super

<details>
<summary>Answer</summary>
B — Producer Extends, Consumer Super. Use `? extends T` for producers (read) and `? super T` for consumers (write).
</details>

## Question 3

Which of the following is NOT a valid wildcard in Java?

A) `?`
B) `? extends Number`
C) `? super String`
D) `? extends T super Number`

<details>
<summary>Answer</summary>
D — A wildcard cannot have both extends and super bounds simultaneously.
</details>

## Question 4

What is type erasure?

A) The JVM deleting unused types at runtime
B) The compiler removing generic type information during compilation
C) The garbage collector removing unused objects
D) A debugging technique for type errors

<details>
<summary>Answer</summary>
B — Type erasure is the compiler removing generic type information, replacing type parameters with their bounds or Object.
</details>

## Question 5

Which of the following is NOT allowed due to type erasure?

A) `new ArrayList<String>()`
B) `List<String> variable`
C) `new T()`
D) `List<String>.class`

<details>
<summary>Answer</summary>
C and D — Cannot instantiate type parameters (`new T()`) and cannot get class of parameterized type (`List<String>.class`). Both are erased at runtime.
</details>

## Question 6

What does the diamond operator `<>` do?

A) Creates a diamond-shaped object
B) Tells the compiler to infer type arguments
C) Indicates a raw type
D) Creates a new generic array

<details>
<summary>Answer</summary>
B — The diamond operator `<>` instructs the compiler to infer the type arguments from the context.
</details>

## Question 7

Is `List<Integer>` a subtype of `List<Number>`?

A) Yes, because Integer extends Number
B) No, because generics are invariant
C) Yes, because of wildcard capture
D) No, because of type erasure

<details>
<summary>Answer</summary>
B — Generics are invariant. `List<Integer>` is NOT a subtype of `List<Number>`. Use `List<? extends Number>` for covariance.
</details>

## Question 8

What is a bridge method?

A) A method that connects two classes
B) A synthetic method generated by the compiler to preserve polymorphism with generics
C) A method in the Collections utility class
D) A method that bridges checked and unchecked exceptions

<details>
<summary>Answer</summary>
B — A bridge method is a synthetic method the compiler generates when a generic interface is implemented, ensuring correct polymorphic dispatch.
</details>

## Question 9

Which annotation is used to suppress unchecked warnings?

A) `@Override`
B) `@FunctionalInterface`
C) `@SuppressWarnings("unchecked")`
D) `@SafeVarargs`

<details>
<summary>Answer</summary>
C — `@SuppressWarnings("unchecked")` suppresses unchecked cast warnings. `@SafeVarargs` suppresses warnings for varargs with generics.
</details>

## Question 10

What is heap pollution?

A) Memory leaks from unused objects
B) A variable of a parameterized type referring to an object not of that type
C) The garbage collector running too frequently
D) Stack overflow from recursive generics

<details>
<summary>Answer</summary>
B — Heap pollution occurs when a variable of a parameterized type refers to an object not of that type, often via raw type or unchecked cast.
</details>
