# Reflection Prompts

## Knowledge Check
1. Without looking, write down the key interfaces and classes. How many did you remember?
2. Explain the concepts to a non-technical stakeholder. What analogies worked?
3. List the top three benefits and give a concrete example of each.

## Deep Understanding
4. What trade-offs are involved? When might simpler code be appropriate?
5. How does this interact with Java's type system?
6. Compare with alternative approaches in other languages.

## Practical Application
7. Look at code you wrote recently. How many principles does it follow?
8. Refactor a piece of your code to better follow best practices.
9. Apply the patterns to a new feature. Did it help?

## Problem Solving
10. Design a solution for a real-time data processing pipeline.
11. How would you handle cross-cutting concerns (logging, metrics, transactions)?
12. What testing strategy would you use?

## Meta-Learning
13. What aspects do you find most challenging?
14. When have you seen these patterns misapplied?
15. How would you teach these concepts to a junior developer?
