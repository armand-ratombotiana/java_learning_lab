# Interview Questions — Functional Programming

## Beginner
1. What is functional programming? How is it different from OOP?
2. What is a pure function? Why does it matter?
3. What is immutability and how does Java support it (records, final, Collections.unmodifiableList)?

## Intermediate
4. Explain Optional — when should you use it and when should you avoid it?
5. What are the monad laws? Does Optional satisfy them?
6. What is function composition in Java? How do `andThen` and `compose` differ?

## Advanced
7. "Why does Java not have true algebraic data types or pattern matching like Haskell?"
8. Design a functional approach to error handling without exceptions.
9. How would you implement a `Try<T>` monad in Java? What are its trade-offs compared to checked exceptions?

## Problem-Solving
10. Implement `memoize` for any pure function.
11. Given a list of `Optional<Integer>`, sum all present values functionally.
12. "I have a deeply nested null-checking if-block. How would you refactor it using Optional?"
