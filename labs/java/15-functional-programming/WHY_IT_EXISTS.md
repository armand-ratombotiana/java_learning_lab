# Why Functional Programming Exists

## Managing Complexity
Imperative programming mixes state and logic, making it hard to reason about. Functional programming separates data from behaviour.

## Parallelism & Concurrency
Pure functions have no shared mutable state — they are trivially safe to parallelise.

## Testability
Pure functions are easy to test — no mocking, no setup.

## Mathematical Rigour
Functional programs correspond more directly to mathematical functions, enabling formal verification.

## Java's Adoption
Java added functional features (lambdas, streams, Optional) to meet the demand for more expressive, safer abstractions — catching up with Scala, Kotlin, and Clojure on the JVM.
