# Testing — Reflection Prompts

## Section 1: Knowledge Check

1. Write down the complete JUnit 5 lifecycle annotation set from memory. Which ones did you miss?
2. Explain the difference between when(x).thenReturn(y) and doReturn(y).when(x).method().
3. List the testing pyramid layers and describe what each layer tests.

## Section 2: Deep Understanding

4. Why does TDD lead to better design? What specific design principles does it enforce?
5. What is the trade-off between mocking real objects and using real implementations in tests?
6. How do parameterized tests improve test maintainability? When are they overkill?

## Section 3: Practical Application

7. Look at a project you maintain. What is the test coverage? Where are the gaps?
8. Refactor a set of test methods into parameterized tests. Did it improve readability?
9. Apply TDD to a small feature. Did the test-first approach change your implementation?

## Section 4: Test Design

10. Design a test suite for a banking application's transfer function. What unit, integration, and E2E tests would you write?
11. What mocks would you create for a microservice that calls three other services?
12. How would you test an asynchronous message processing pipeline?

## Section 5: Meta-Learning

13. What is your personal testing workflow? How could you improve it?
14. When have tests saved you from a production bug? When have tests failed to catch a bug?
15. How do you convince a team that doesn't test to adopt testing practices?
