# Lab 30: JVM Internals

## Overview
The Java Virtual Machine (JVM) is the runtime engine that executes Java bytecode. This lab explores JVM architecture, memory management, garbage collection, bytecode, class loading, JIT compilation, and performance tuning.

## Key Concepts
- JVM architecture (stack-based execution engine)
- Class loading and linking
- Runtime data areas (heap, stack, method area, PC registers, native method stacks)
- Garbage collection algorithms (G1, ZGC, Shenandoah)
- JIT compilation (C1, C2, tiered compilation)
- Bytecode instructions and verification
- JVM tuning and monitoring
