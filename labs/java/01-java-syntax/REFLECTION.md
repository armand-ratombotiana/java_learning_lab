# Java Syntax — Reflection Prompts

## Section 1: Knowledge Check

1. Without looking at any reference, write down the complete list of Java keywords you can remember. How many did you get? Which ones did you forget?

2. Explain the difference between `final`, `finally`, and `finalize()` to a fellow developer. Each starts with "final" but has different meanings.

3. Write the operator precedence table from memory. Which levels do you consistently get wrong?

4. Describe the Java naming conventions for: classes, methods, variables, constants, packages, and type parameters. Give a real example of each.

## Section 2: Deep Understanding

5. Why does Java not support operator overloading for user-defined types? Do you agree with this design decision? What are the trade-offs?

6. Why does Java use `boolean` as a separate type instead of using `0`/`1` like C? How does this affect safety?

7. What is the rationale behind `String[] args` instead of just a simpler alternative? Why does `main` need to be `public static`?

8. Consider: Java added `var` in Java 10 but restricts it to local variables. Why not allow it for fields and method returns? What would the trade-offs be?

## Section 3: Practical Application

9. Look at code you wrote last week. How many naming convention violations do you find? How many places could benefit from `var`?

10. Write a method using only the ternary operator (no if-else). Is it more readable? Less readable? When is ternary appropriate?

11. Refactor a `switch` statement you've written into a `switch` expression. Does it improve readability?

12. Convert a method with nested blocks (if inside for inside if) into a flatter structure. Does the new version have fewer syntax pitfalls?

## Section 4: Self-Assessment

13. Rate your understanding of Java syntax from 1 (struggling) to 5 (expert). For each area below:
    - Keywords and their meanings
    - Operator precedence
    - Naming conventions
    - Comment styles
    - Literal formats
    - Access modifiers
    - Block structure and scoping

14. What is the most confusing syntax feature for you? Why? What would help clarify it?

15. Have you ever caused a production bug due to syntax confusion? What was the root cause? How could it have been prevented?

16. Create a personal "syntax cheat sheet" — what are the 5 syntax rules you most frequently need to remind yourself about?

## Section 5: Next Steps

17. What Java syntax feature added in Java 8 or later do you find most valuable? Why?

18. Are there any syntax features from other languages (Python, Kotlin, C#, TypeScript) that you wish Java had? Why?

19. How would you teach Java syntax to a complete programming beginner? What analogies would you use?

20. What aspect of Java syntax do you want to explore more deeply after this lab?
