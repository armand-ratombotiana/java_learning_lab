# Flashcards — Annotations

**Q:** What is an annotation?  
**A:** Metadata embedded in source code, storable in class files, accessible at runtime.

**Q:** What is `@Retention(RUNTIME)`?  
**A:** Makes annotation available via reflection at runtime.

**Q:** What is `@Target`?  
**A:** Specifies which code elements the annotation can be applied to.

**Q:** How do you define a custom annotation?  
**A:** `@interface MyAnnotation { }`

**Q:** What is `@Override`?  
**A:** Ensures a method overrides a superclass/interface method.

**Q:** What is `@FunctionalInterface`?  
**A:** Indicates an interface has exactly one abstract method.

**Q:** What is `@Deprecated`?  
**A:** Marks element as obsolete, compiler generates warning.

**Q:** What is `@Repeatable`?  
**A:** Allows applying the same annotation multiple times to an element.

**Q:** What is an annotation processor?  
**A:** A compile-time tool that processes annotations and can generate code.

**Q:** What is `@SuppressWarnings`?  
**A:** Suppresses specified compiler warnings.
