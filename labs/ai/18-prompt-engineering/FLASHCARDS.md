# Prompt Engineering - FLASHCARDS

### Card 1
**Q:** What is few-shot learning in prompts?
**A:** Including examples in the prompt to guide the model's behavior

### Card 2
**Q:** What is chain-of-thought (CoT) prompting?
**A:** Asking model to show reasoning steps before answer, improves complex reasoning

### Card 3
**Q:** Why use role-based prompts?
**A:** Assigning a persona helps model adopt specific tone and expertise level

### Card 4
**Q:** What is prompt template?
**A:** Reusable prompt structure with placeholders for variables

### Card 5
**Q:** What is self-consistency?
**A:** Generate multiple solutions and pick most common answer

### Card 6
**Q:** What makes good system prompt?
**A:** Clear role definition, specific guidelines, output format instructions

### Card 7
**Q:** Why include constraints in prompts?
**A:** Tell model what NOT to do, reduces unwanted outputs

### Card 8
**Q:** What is zero-shot prompting?
**A:** Task description without examples, model uses prior knowledge

### Card 9
**Q:** How improve prompt iteratively?
**A:** Test, analyze failures, adjust prompt, retest

### Card 10
**Q:** What is prompt injection?
**A:** Malicious input designed to override original instructions

### Card 11
**Q:** Why use delimiters in prompts?
**A:** Clearly separate different parts (instructions, context, examples)

### Card 12
**Q:** What is prompt tuning?
**A:** Training lightweight prompts that can be optimized for tasks
