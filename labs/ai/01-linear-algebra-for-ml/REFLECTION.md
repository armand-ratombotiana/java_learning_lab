# Linear Algebra for ML - Reflection Questions

## Understanding Check

### R1: Core Concepts Mastery
1. Can you explain why matrix multiplication is not commutative? Give an example where AB and BA give different results.
2. Describe in your own words what eigenvalues and eigenvectors represent geometrically.
3. Why is SVD preferred over eigendecomposition in many ML applications?

### R2: Connection to ML
1. How does the rank of a matrix relate to overfitting in ML models?
2. In what ways does PCA use linear algebra concepts you've learned?
3. How does regularization (ridge/lasso) change the linear algebra of the solution?

### R3: Computational Considerations
1. Why is numerical stability important in linear algebra computations?
2. When would you choose iterative methods over direct methods?
3. How does the condition number affect the accuracy of your results?

## Practical Applications

### R4: Real-World Scenarios
1. Describe a problem where you would use truncated SVD.
2. In what situations would Cholesky decomposition be preferred over LU?
3. How would you handle a singular matrix in a production system?

### R5: Implementation Decisions
1. What data structures would you use for sparse matrices? Why?
2. How would you design a linear algebra library for ML?
3. What testing would you implement for numerical stability?

## Advanced Topics

### R6: Deeper Understanding
1. How does the spectral radius relate to convergence of iterative methods?
2. What is the relationship between SVD and the pseudoinverse?
3. How do singular values relate to the "energy" or information in a matrix?

### R7: Optimization
1. How does vectorization improve performance in linear algebra operations?
2. What are the trade-offs between time and space complexity in matrix operations?
3. How would you parallelize matrix decompositions?
