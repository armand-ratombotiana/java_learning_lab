# Exercises: EBS Technical Architecture

## Exercise 1: Basic Implementation

Write a Java method that simulates EBS Technical Architecture processing with input validation, processing logic, and result generation.

## Exercise 2: Configuration Management

Extend the simulator to support configuration profiles loaded from a simulated database (Map-based).

## Exercise 3: Error Handling

Add transaction rollback support. Simulate a mid-process failure and verify partial updates are rolled back.

## Exercise 4: Performance Optimization

Implement batch processing. Compare single-record vs batch performance.

## Exercise 5: Integration

Create an integration layer connecting with a simulated Workflow engine.

## Exercise 6: Testing

Write JUnit 5 parameterized tests for edge cases: null inputs, boundary values, and concurrent access.

## Exercise 7: Security

Add function security checks simulating FND_FORM_FUNCTIONS access control.

## Exercise 8: Audit

Implement audit trail generation recording user, timestamp, and action for every transaction.

## Exercise 9: Multi-Org

Add MOAC support ensuring data isolation between operating units.

## Exercise 10: Full Stack

Combine all previous exercises into a complete simulation with CLI interface.
