# Why PostgreSQL Exists

PostgreSQL exists as a **free, open-source, feature-rich relational database** that rivals commercial databases.

## History Summary
- Started as POSTGRES (UC Berkeley, 1986) by Michael Stonebraker
- Post-Ingres project (after commercializing Ingres)
- SQL support added in Postgres95 (1994)
- Renamed PostgreSQL in 1996
- Today: one of the most advanced open-source databases

## Why It Exists Alongside MySQL
- PostgreSQL: "The world's most advanced open-source database"
- Stronger SQL standard compliance
- More advanced features (CTEs, window functions, extensions)
- Better concurrency (MVCC without messy locking)
- Extensibility – custom types, operators, indexes

## Why Java Developers Choose PostgreSQL
- Excellent JDBC driver (Type 4, pure Java)
- Full JPA support with Hibernate PostgreSQL dialect
- JSON/JSONB for hybrid relational + document storage
- MVCC performance for OLTP workloads
- Rich extension ecosystem for specialized needs
