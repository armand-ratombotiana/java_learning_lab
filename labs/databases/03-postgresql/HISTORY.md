# History of PostgreSQL

## Timeline

| Year | Event |
|---|---|
| 1986 | POSTGRES project begins at UC Berkeley (Stonebraker) |
| 1989 | POSTGRES 1.0 release |
| 1994 | Postgres95 (SQL support added by Andrew Yu, Jolly Chen) |
| 1996 | PostgreSQL 6.0 (name change, community release) |
| 1997 | PostgreSQL 6.1 (MVCC) |
| 2005 | PostgreSQL 8.0 (native Windows support) |
| 2006 | PostgreSQL 8.1 (two-phase commit) |
| 2010 | PostgreSQL 9.0 (hot standby, streaming replication) |
| 2011 | PostgreSQL 9.1 (extension management, pg_stat_statements) |
| 2012 | PostgreSQL 9.2 (JSON support, index-only scans) |
| 2014 | PostgreSQL 9.4 (JSONB, materialized views, logical decoding) |
| 2015 | PostgreSQL 9.5 (BRIN indexes, row-level security, UPSERT) |
| 2016 | PostgreSQL 9.6 (parallel query) |
| 2017 | PostgreSQL 10 (logical replication, declarative partitioning) |
| 2018 | PostgreSQL 11 (parallel index build, JIT compilation) |
| 2019 | PostgreSQL 12 (SQL/JSON path, generated columns) |
| 2020 | PostgreSQL 13 (incremental sort, parallel VACUUM) |
| 2021 | PostgreSQL 14 (pipeline mode, connection defaults) |
| 2022 | PostgreSQL 15 (MERGE, UNIQUE NULLS NOT DISTINCT) |
| 2023 | PostgreSQL 16 (logical replication improvements, performance) |
| 2024 | PostgreSQL 17 (further performance, SQL/JSON enhancements) |
