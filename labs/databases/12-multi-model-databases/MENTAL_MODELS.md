# Mental Models: Polyglot Persistence

## Toolbox
Polyglot persistence is like a toolbox. You don't fix every problem with one tool. Hammer for nails (relational), screwdriver for screws (document), wrench for bolts (graph). Each tool is optimized for its job. A Swiss Army knife (multi-model DB) handles basic needs but may not excel at specialized tasks.

## Lego Blocks
Different data types are different Lego shapes. Relational blocks are fixed rectangular bricks (rigid schema). Document blocks are flexible, interlocking pieces (dynamic shape). Graph blocks have connectors on every face (traversal). Build each part of your creation with the right block type.

## Warehouse vs Boutique
A big-box store (single-model DB) carries everything but may not have the best quality in each category. A polyglot approach (boutique shops) gets the best bread from the bakery, best meat from the butcher, best produce from the greengrocer – but you need to manage multiple vendor relationships.

## Special Forces
Each database type is a specialist:
- Relational = Accountant (accuracy, rules)
- Document = Writer (flexibility, creativity)
- Graph = Detective (connections, patterns)
- Key-Value = Courier (fast, simple, no questions asked)
