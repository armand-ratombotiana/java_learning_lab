# History: Multi-Model & Polyglot Persistence

- **1970s-2000s** – Relational dominance (one model to rule them all)
- **2006** – Amazon Dynamo paper (key-value, eventual consistency)
- **2008** – Neo4j (native graph database), CouchDB (document)
- **2009** – MongoDB 1.0 (document), Cassandra (columnar)
- **2010** – Redis (key-value with rich data structures)
- **2012** – ArangoDB launches as native multi-model (doc + graph + key-value)
- **2015** – Martin Fowler popularizes "polyglot persistence" term
- **2017** – Microsoft Cosmos DB (multi-model, globally distributed)
- **2019** – Amazon DocumentDB, Neptune (graph)
- **2021** – SingleStore, Yugabyte support multi-model
- **2023** – Vector databases surge (Pinecone, Weaviate, pgvector)
- **2024+** – AI-native databases, unified query engines (Dremio, Trino)
