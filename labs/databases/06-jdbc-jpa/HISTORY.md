# History: JDBC & JPA

## JDBC Timeline
- **1997**: JDBC 1.0 (java.sql package)
- **1999**: JDBC 2.0 (ResultSet updates, batch updates, connection pooling)
- **2001**: JDBC 3.0 (savepoints, connection pooling improvements)
- **2006**: JDBC 4.0 (auto-loading drivers, SQL/XML support)
- **2013**: JDBC 4.1 (try-with-resources, return generated keys)
- **2017**: JDBC 4.3 (JDK 9, reactive streams pilot)
- **2021**: JDBC 5.0 (Jakarta EE 9, javax→jakarta migration)

## Hibernate Timeline
- **2001**: Hibernate 1.0 created by Gavin King
- **2003**: Hibernate 2.0 — HQL, Criteria API, caching
- **2005**: Hibernate 3.0 — Annotations, second-level cache, filters
- **2006**: Hibernate became JPA 1.0 reference implementation
- **2010**: Hibernate 3.6 — JPA 2.0 compatibility
- **2013**: Hibernate 4.3 — JPA 2.1 (Stored Procedures, bulk updates)
- **2017**: Hibernate 5.2 — Java 8 features, streaming queries
- **2021**: Hibernate 5.5 — Jakarta EE 9 migration
- **2022**: Hibernate 6.1 — JDK 17, HQL type-safe queries
- **2024**: Hibernate 6.5 — Query by natural ID, attribute converters

## JPA Timeline
- **2006**: JPA 1.0 (JSR 220) — EntityManager, annotations, JPQL
- **2009**: JPA 2.0 (JSR 317) — Criteria API, collection of basics, pessimistic locking
- **2013**: JPA 2.1 (JSR 338) — Stored procedures, bulk updates, schema generation
- **2019**: JPA 2.2 (maintenance) — Java 8 date/time types, repeatable annotations
- **2022**: JPA 3.0 (Jakarta Persistence 3.0) — javax→jakarta migration
- **2023**: JPA 3.1 — Query result streaming, UUID support
