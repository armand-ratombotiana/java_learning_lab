# History: Event-Driven Architecture

## Origins
Event-driven architecture traces back to early distributed systems research. The publish-subscribe pattern was formalized in the 1990s.

## Evolution

### 1990s
TIBCO Rendezvous and IBM MQ provided early message-oriented middleware. Publish-subscribe pattern became standard.

### 2000s
Enterprise Service Bus (ESB) pattern emerged. Event-driven integration became mainstream in enterprise architecture.

### 2011
Martin Fowler and Greg Young popularized Event Sourcing and CQRS. The Event Store database was released.

### 2014
Apache Kafka became a de facto standard for event streaming. Event-driven microservices gained adoption.

### 2020s
Event sourcing becomes mainstream. Serverless event-driven architectures emerge.

## Key Milestones
- 1990s: Message-oriented middleware
- 2000s: Enterprise Service Bus
- 2011: Event Sourcing introduced
- 2014: Apache Kafka 1.0
- 2018: AWS EventBridge
- 2020: Event sourcing in mainstream practice
