# Hexagonal Architecture Quiz

## Question 1
What is a port in hexagonal architecture?
- a) A network endpoint
- b) An interface defining interaction with the core
- c) A database connection
- d) A REST API

**Answer: b)**

## Question 2
What is the dependency direction in hexagonal architecture?
- a) Adapters depend on domain
- b) Domain depends on adapters
- c) Bidirectional dependencies
- d) No dependencies at all

**Answer: a)**

## Question 3
What is a driving adapter?
- a) An adapter that interacts with databases
- b) An adapter that initiates interaction with the core
- c) An adapter that sends events
- d) An adapter for logging

**Answer: b)**

## Question 4
What should NOT be in the domain layer?
- a) Business logic
- b) Value objects
- c) Framework annotations
- d) Domain events

**Answer: c)**

## Question 5
What is the benefit of ports and adapters for testing?
- a) No testing needed
- b) Domain can be tested without infrastructure
- c) Tests run faster in production
- d) More test coverage

**Answer: b)**

## Question 6
Who introduced Hexagonal Architecture?
- a) Robert C. Martin
- b) Martin Fowler
- c) Alistair Cockburn
- d) Eric Evans

**Answer: c)**
