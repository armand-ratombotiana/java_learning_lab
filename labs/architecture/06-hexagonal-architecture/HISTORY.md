# History of Hexagonal Architecture

## Origins
- **2005** - Alistair Cockburn introduces Hexagonal Architecture
- **2008** - Cockburn publishes the "Hexagonal Architecture" article
- Also known as "Ports and Adapters" pattern

## Evolution
- **2008-2012** - Adoption in DDD community
- **2013** - Popularized with microservices architecture
- **2015** - "Clean Architecture" by Robert C. Martin (similar principles)
- **2017** - ArchUnit library for Java architectural testing
- **2019** - Spring Boot 2.x fully compatible with hexagonal structure
- **2020+** - Mainstream adoption for domain-centric microservices

## Related Patterns
- **Onion Architecture** (Jeffrey Palermo, 2008)
- **Clean Architecture** (Robert C. Martin, 2012)
- **Ports and Adapters** (Alistair Cockburn)

## Java Adoption
- Early adopter: Java ecosystem with strong interfaces
- Spring's dependency injection supports adapter switching
- ArchUnit enables architectural rule enforcement
- Maven/Gradle module structure supports hexagonal packages
