# History: Strangler Fig Pattern

## Origins
The Strangler Fig pattern was first described by Martin Fowler in 2004. It was inspired by the strangler fig tree, which grows around a host tree and eventually replaces it entirely.

## Evolution

### Early 2000s
The concept emerged from large-scale system migration projects. Organizations with monolithic legacy systems needed a way to migrate incrementally.

### 2010s
Cloud migration popularized the pattern as organizations moved from on-premise to cloud infrastructure. The pattern became a standard approach for legacy modernization.

### 2020s
The pattern evolved to include advanced techniques like feature flags, branch by abstraction, and automated migration coordination tools.

## Key Milestones
- 2004: Martin Fowler coins the term on his blog
- 2008: First documented large-scale strangulation at Amazon
- 2012: Netflix adopts the pattern for their cloud migration
- 2015: Feature flag tools mature (LaunchDarkly, Unleash)
- 2018: Branch by abstraction formalized as a refactoring pattern
- 2021: Automated strangulation tools emerge
