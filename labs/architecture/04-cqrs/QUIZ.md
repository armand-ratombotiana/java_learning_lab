# CQRS Quiz

## Question 1
What does CQRS stand for?
- a) Command Query Responsibility Segregation
- b) Concurrent Query Response System
- c) Command Query Read Separation
- d) Complete Query Resolution Strategy

**Answer: a)**

## Question 2
What is a command in CQRS?
- a) A request to read data
- b) An expression of intent to change state
- c) A database query
- d) An event notification

**Answer: b)**

## Question 3
What is a projection in CQRS?
- a) A database view
- b) A component that updates read models from events
- c) A command handler
- d) An API endpoint

**Answer: b)**

## Question 4
When should you use CQRS?
- a) For every application
- b) When read and write workloads have different characteristics
- c) Only with event sourcing
- d) For simple CRUD apps

**Answer: b)**

## Question 5
What is the relationship between CQRS and Event Sourcing?
- a) CQRS requires Event Sourcing
- b) Event Sourcing requires CQRS
- c) They are independent but complementary patterns
- d) They are the same thing

**Answer: c)**

## Question 6
What is a common challenge with CQRS?
- a) Performance is always worse
- b) Eventual consistency between read and write models
- c) Cannot use SQL databases
- d) Requires microservices

**Answer: b)**
