# Event Sourcing Quiz

## Question 1
What is the primary storage in Event Sourcing?
- a) Current state in database
- b) Sequence of events
- c) Both events and current state
- d) Temporary files

**Answer: b)**

## Question 2
What is a snapshot in Event Sourcing?
- a) A screenshot of the UI
- b) A saved aggregate state at a point in time
- c) A database backup
- d) An event type

**Answer: b)**

## Question 3
How do you rebuild aggregate state in Event Sourcing?
- a) Read from database
- b) Replay events in order
- c) Load from cache
- d) Call external API

**Answer: b)**

## Question 4
What happens when an event schema changes?
- a) Events are updated in place
- b) Old events cannot be deserialized
- c) Use upcasting to migrate old events
- d) Delete old events

**Answer: c)**

## Question 5
What is the relationship between Event Sourcing and CQRS?
- a) ES requires CQRS
- b) CQRS requires ES
- c) They are complementary but independent
- d) They are competing patterns

**Answer: c)**

## Question 6
What is a key benefit of Event Sourcing?
- a) Simpler code
- b) Complete audit trail
- c) Better performance
- d) Less storage

**Answer: b)**
