# History: Advanced Strangler Fig

## Origins
The Advanced Strangler Fig pattern builds on the original strangler fig concept, adding sophisticated decomposition, database splitting, and async migration techniques.

## Evolution

### 2015-2017
As organizations gained experience with basic strangulation, they encountered challenges with database splitting and large-scale coordination.

### 2018
Event Storming and Domain-Driven Design provided tools for identifying service boundaries. Database splitting strategies were formalized.

### 2020
Change Data Capture (CDC) tools like Debezium matured. Async migration patterns became viable for zero-downtime migrations.

### 2022
Automated migration coordination tools emerged. Feature flag management platforms integrated with migration workflows.

## Key Milestones
- 2015: Database strangulation patterns documented
- 2017: Event Storming for decomposition
- 2019: Debezium CDC for data migration
- 2021: Automated migration orchestration
