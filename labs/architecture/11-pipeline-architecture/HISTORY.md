# History of Pipeline Architecture

## Origins
- **1960s** - Unix pipes concept (Doug McIlroy)
- **1972** - Unix pipe operator "|" introduced
- **1995** - Chain of Responsibility pattern (GoF)

## Software Evolution
- **2004** - Java Streams proposed (not implemented until Java 8)
- **2005** - Apache Camel for enterprise integration pipelines
- **2007** - Spring Batch for batch processing pipelines
- **2010** - Jenkins CI/CD pipeline concept
- **2014** - Java 8 Streams API introduces functional pipelines
- **2015** - Reactive Streams for async pipelines
- **2018** - Apache NiFi for data flow pipelines
- **2020+** - Pipeline-as-code (GitHub Actions, GitLab CI)

## Java Evolution
- Java 8 Streams (2014) - Functional data pipelines
- Reactor (2015) - Reactive pipelines
- Spring Cloud Data Flow (2016) - Stream and batch pipelines
- Spring Batch 4.x (2017) - Batch processing pipelines
- Java 9 Flow API (2017) - Reactive streams standard
