# History of Six-Port Architecture

## Origins
- **2015-2017** - Evolution from Hexagonal Architecture
- **2017** - Formalized as "Six-Port Architecture" by various practitioners
- **2018** - Adoption in microservices ecosystem

## Evolution
- **2015** - Hexagonal Architecture widely adopted
- **2016** - Practitioners note ambiguity in port categorization
- **2017** - Six distinct port types emerge as convention
- **2018** - Six-Port naming conventions spread
- **2019** - Tools and templates for Six-Port architecture
- **2020+** - Used alongside DDD and microservices

## Key Influences
- Hexagonal Architecture (Cockburn)
- Clean Architecture (Martin)
- DDD (Evans)
- Enterprise Integration Patterns (Hohpe)
- Event-Driven Architecture

## Java Experience
- Spring Boot ecosystem supports Six-Port naturally
- Interfaces as ports, @Component as adapters
- Convention over configuration for naming
- Event-driven support via Spring Cloud Stream
