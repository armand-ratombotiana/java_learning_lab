# History: API Composition

## Origins
API composition emerged naturally as microservices architectures scaled. The need to aggregate data from multiple services created the API composition pattern.

## Evolution

### 2010-2014
Early microservices adopters built custom aggregation layers. The API Gateway pattern emerged as a solution for cross-cutting concerns.

### 2015
GraphQL was released by Facebook, providing declarative data fetching. GraphQL became a popular approach for API composition.

### 2017
GraphQL federation was introduced, enabling schema composition across multiple services.

### 2019
BFF composition patterns formalized for client-specific aggregation.

## Key Milestones
- 2015: GraphQL public release
- 2016: API Gateway pattern mainstream adoption
- 2017: Apollo Federation specification
- 2018: Netflix GraphQL federation
- 2020: BFF composition standardized
