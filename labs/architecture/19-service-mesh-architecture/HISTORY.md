# History: Service Mesh

## Origins
Service mesh evolved from the need to manage service-to-service communication in microservices. It emerged at the intersection of proxy technology and container orchestration.

## Evolution

### 2014
Netflix developed the Prana sidecar. Lyft began work on Envoy proxy.

### 2016
Buoyant launched Linkerd, the first dedicated service mesh. Envoy was open-sourced by Lyft.

### 2017
Istio was announced jointly by Google, IBM, and Lyft. It became the most popular service mesh platform.

### 2018
Linkerd 2.0 rewritten in Rust for better performance. CNCF accepted both Envoy and Linkerd.

### 2020s
Service mesh becomes standard infrastructure. Cloud providers offer managed service meshes. Sidecar-less mesh alternatives emerge.

## Key Milestones
- 2014: Netflix Prana
- 2016: Linkerd and Envoy release
- 2017: Istio announcement
- 2018: Linkerd 2.0, CNCF acceptance
- 2019: AWS App Mesh
- 2021: Istio graduates CNCF
