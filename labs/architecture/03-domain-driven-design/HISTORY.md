# History of Domain-Driven Design

## Origins
- **2003** - Eric Evans publishes "Domain-Driven Design: Tackling Complexity in the Heart of Software"
- **2004** - First DDD conference talks and community formation

## Key Milestones
- **2006** - Jimmy Nilsson publishes "Applying Domain-Driven Design and Patterns"
- **2009** - Vaughn Vernon starts the DDD community site dddcommunity.org
- **2013** - Vaughn Vernon publishes "Implementing Domain-Driven Design"
- **2014** - Event Storming introduced by Alberto Brandolini
- **2015** - CQRS/ES patterns gain DDD adoption
- **2019** - "Domain-Driven Design Distilled" by Vaughn Vernon

## Java Ecosystem Evolution
- **2006** - Hibernate/Java Persistence API for ORM
- **2010** - Spring framework adoption for DDD
- **2014** - Axon Framework gains popularity for DDD+CQRS
- **2018** - JPA 2.2 with better value object support
- **2020** - Record types (Java 14+) for value objects
- **2023** - Spring Modulith for modular monolith with DDD

## Modern Evolution
- DDD meets Microservices (each bounded context = potential service)
- Event Storming becomes standard domain discovery technique
- DDD for event-driven architectures
- Example Mapping for specification by example
