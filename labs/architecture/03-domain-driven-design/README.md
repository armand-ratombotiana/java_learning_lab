# Domain-Driven Design (DDD)

## Overview
Domain-Driven Design is an approach to software development that emphasizes collaboration between technical and domain experts to create models that reflect complex business domains. DDD provides a set of strategic and tactical patterns for building domain-rich software.

## Topics Covered
- Ubiquitous language and strategic design
- Bounded contexts and context mapping
- Aggregates, entities, and value objects
- Domain events and integration events
- Repositories and domain services
- Application services and infrastructure concerns
- Event storming for domain discovery

## Java/Spring Stack
- Spring Data JPA for persistence
- Axon Framework for domain events
- Jackson for domain event serialization
- ArchUnit for architectural testing
- JPA/Hibernate for aggregate persistence
