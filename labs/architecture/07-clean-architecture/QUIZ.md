# Clean Architecture Quiz

## Question 1
What is the primary rule of Clean Architecture?
a) All layers can depend on each other
b) Source code dependencies point inward
c) Entities depend on frameworks
d) Adapters contain business logic

**Answer: b) Source code dependencies point inward**

## Question 2
Which layer contains enterprise business rules?
a) Framework layer
b) Interface adapter layer
c) Entity layer
d) Use case layer

**Answer: c) Entity layer**

## Question 3
What is a use case interactor?
a) A database access object
b) An application-specific business rule
c) A REST controller
d) A UI component

**Answer: b)**

## Question 4
What is the purpose of a presenter?
a) To handle HTTP requests
b) To format output data for the delivery mechanism
c) To access databases
d) To validate input

**Answer: b)**

## Question 5
Who created Clean Architecture?
a) Martin Fowler
b) Eric Evans
c) Robert C. Martin
d) Alistair Cockburn

**Answer: c) Robert C. Martin**

## Question 6
What does "screaming architecture" mean?
a) Architecture that is loud
b) Project structure that reveals system purpose
c) Architecture with errors
d) Performance optimization

**Answer: b)**
