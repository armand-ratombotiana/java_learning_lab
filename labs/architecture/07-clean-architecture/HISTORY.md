# History of Clean Architecture

## Origins
- **2012** - Robert C. Martin (Uncle Bob) publishes "Clean Architecture"
- **2012** - Blog post "The Clean Architecture" on blog.8thlight.com

## Influences
- **Hexagonal Architecture** by Alistair Cockburn (2005)
- **Onion Architecture** by Jeffrey Palermo (2008)
- **Domain-Driven Design** by Eric Evans (2003)
- **SOLID Principles** by Robert C. Martin (2000s)

## Evolution
- **2012-2015** - Clean Architecture gains popularity
- **2015** - Widely adopted with microservices movement
- **2017** - "Clean Architecture: A Craftsman's Guide" book published
- **2018** - ArchUnit enables automated architecture verification
- **2019** - Spring Boot 2.x aligns with clean architecture patterns
- **2023** - "Get Your Hands Dirty on Clean Architecture" by Tom Hombergs

## Java Implementation Evolution
- Early: Manual package separation (entities, usecases, gateways)
- 2017+: ArchUnit for compile-time checks
- 2020+: Clean Architecture project templates and starters
- 2023+: Spring Modulith with clean architecture support
