# History: Backend for Frontend

## Origins
The Backend for Frontend pattern was first described by Phil Calçado and colleagues at SoundCloud in 2015. It emerged from the challenges of supporting multiple client types with a single API.

## Evolution

### 2010-2014
Mobile applications created demand for optimized APIs. Organizations struggled with monolithic APIs that served both web and mobile clients with different needs.

### 2015
SoundCloud publicly described the BFF pattern. The concept gained rapid adoption in the microservices community.

### 2017-2019
GraphQL emerged as a complementary technology for BFFs. Companies like Netflix and Twitter adopted BFF patterns for their mobile backends.

### 2020s
BFF becomes a standard pattern in cloud-native architectures. API Gateways evolve to support BFF-specific routing and management.

## Key Milestones
- 2015: SoundCloud article on BFF pattern
- 2016: BFF adopted in ThoughtWorks Technology Radar
- 2018: GraphQL federation introduced for BFF composition
- 2020: BFF becomes standard in cloud-native architectures
