# History of Event-Driven Architecture

## Early Foundations
- **1960s** - Message queuing systems (IBM MQ origins)
- **1980s** - Publish/subscribe systems emerge
- **1990s** - CORBA Event Service, JMS specification

## Modern Era
- **2004** - Amazon Simple Queue Service (SQS) launched
- **2007** - RabbitMQ (AMQP implementation) released
- **2011** - Apache Kafka created at LinkedIn
- **2014** - Apache Spark Streaming for real-time processing
- **2016** - Kafka Streams library released
- **2018** - Event sourcing gains mainstream adoption

## Java Ecosystem Evolution
- **JMS** (1999) - Java Message Service standard
- **Spring JMS** - Simplified JMS integration
- **Spring AMQP** - RabbitMQ abstraction
- **Spring Cloud Stream** - Unified messaging abstraction (2016)
- **Spring Kafka** - Kafka support (2017)
- **Spring Modulith** (2023) - Event-driven monolith support

## Key Influences
- Domain-Driven Design by Eric Evans (2003)
- Event Sourcing by Martin Fowler (2005)
- Reactive Manifesto (2014)
- Microservices patterns by Chris Richardson
