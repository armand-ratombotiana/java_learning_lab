# Event-Driven Architecture

## Overview
Event-Driven Architecture (EDA) is a software architecture pattern where components communicate by producing and consuming events. Events represent significant state changes and are typically published to a message broker for consumption by interested parties.

## Topics Covered
- Event-driven patterns and event types
- Event sourcing fundamentals
- Event buses and message brokers (Kafka, RabbitMQ)
- Async communication patterns
- Event-driven microservices integration
- Stream processing with Kafka Streams
- CQRS with event-driven backends

## Java/Spring Stack
- Spring Cloud Stream for event bindings
- Apache Kafka / RabbitMQ as message broker
- Kafka Streams / Spring Cloud Stream for processing
- Axon Framework for event sourcing
- Spring Modulith for event-driven modules

## Prerequisites
- Java 17+
- Spring Boot 3.x
- Basic understanding of messaging systems
