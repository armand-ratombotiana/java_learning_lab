# History of Terraform

## Timeline

| Year | Milestone |
|------|-----------|
| 2014 | **Terraform** v0.1 released by HashiCorp |
| 2015 | v0.3 (state management); v0.4 (modules) |
| 2016 | v0.7 (state locking); v0.8 (HCL v2 syntax) |
| 2017 | Terraform Enterprise; **Terraform Registry** launches |
| 2018 | v0.12 (first-class expressions, rich types) |
| 2019 | Terraform Cloud (remote operations, VCS-driven runs) |
| 2020 | v0.13 (module improvements); **HashiCorp IPO** |
| 2021 | v1.0 (stable, backward compatibility guarantee); HCL JSON support |
| 2022 | v1.2 (updated provider installation); v1.3 (optional attributes) |
| 2023 | v1.4 (remote plan caching); v1.5 (config-driven import) |
| 2024 | v1.7 (removed deprecated features); **HCL v2.0** updates |
| 2024 | **OpenTofu** fork (Linux Foundation; open-source alternative after BSL license change) |

## Key Insight
Terraform's v1.0 release (2021) was a landmark: the first major version promised backward compatibility. Organizations that hesitated to adopt during the 0.x era felt confident committing to Terraform. The 2023 license change (MPL → BSL) spawned OpenTofu — the first major fork in the IaC ecosystem.
