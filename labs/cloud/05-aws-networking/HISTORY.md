# History of AWS Networking

## Timeline

| Year | Milestone |
|------|-----------|
| 2006 | EC2 launched with basic networking (no VPC) — "EC2-Classic" |
| 2009 | **Route 53** launches (DNS + domain registration) |
| 2009 | **ELB** (Elastic Load Balancing) launches — first-gen load balancer |
| 2011 | **VPC** launches — isolated networks within AWS |
| 2013 | VPC becomes default for new accounts; **CloudFront** launches |
| 2014 | **ALB** (Application Load Balancer) — Layer 7, host/path routing |
| 2015 | **VPC Peering** — connect VPCs privately |
| 2016 | **NLB** (Network Load Balancer) — Layer 4, ultra-low latency |
| 2017 | **Transit Gateway** — hub-and-spoke network hub |
| 2018 | VPC Endpoints for S3 and DynamoDB |
| 2019 | AWS PrivateLink — expose services privately across VPCs |
| 2020 | **CloudFront** Lambda@Edge; Global Accelerator |
| 2021 | VPC Lattice — service-to-service networking |
| 2022 | Route 53 Resolver DNS Firewall |
| 2023 | CloudFront Origin Access Control (OAC) |
| 2024 | VPC Lattice cross-account support; CloudFront security updates |

## Key Insight
VPC transformed AWS from a public-cloud-only platform (EC2-Classic) to an enterprise-ready infrastructure. Overnight, enterprises could build isolated networks in the cloud with the same security posture as their on-premise data centers.
