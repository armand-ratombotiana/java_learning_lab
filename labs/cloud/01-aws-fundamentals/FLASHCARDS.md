# AWS Fundamentals - Flashcards

## Key Terms and Concepts

### EC2 (Elastic Compute Cloud)
Virtual servers in the cloud. Provides scalable compute capacity.

### S3 (Simple Storage Service)
Object storage service offering industry-leading scalability, availability, and security.

### IAM (Identity and Access Management)
Service for managing users, groups, roles, and their permissions to AWS resources.

### VPC (Virtual Private Cloud)
Isolated virtual network within AWS with controllable IP addressing and networking components.

### AWS Region
Geographic area containing multiple Availability Zones for deploying infrastructure.

### Availability Zone (AZ)
Isolated data center within a region with independent power, networking, and connectivity.

### Security Group
Virtual firewall for EC2 instances controlling inbound and outbound traffic at instance level.

### NACL (Network Access Control List)
Stateless firewall at subnet level controlling traffic in and out of subnets.

### EBS (Elastic Block Store)
Block storage volumes for use with EC2 instances.

### IAM Policy
JSON document defining permissions for IAM identities (users, groups, roles).

### IAM Role
Identity with permissions that can be assumed by trusted entities (services, users).

### Internet Gateway
Component enabling communication between VPC and the internet.

### NAT Gateway
Network Address Translation device enabling instances in private subnet to access internet.

### S3 Bucket
Container for storing objects in S3, identified by unique global name.

### S3 Lifecycle Policy
Rule defining automatic transition of objects between storage classes.

### S3 Versioning
Feature preserving multiple versions of objects to protect against accidental deletion.

### EC2 Instance Type
Type defining compute resources (vCPU, memory, network performance).

### Reserved Instance
Pricing option offering discount in exchange for 1 or 3 year commitment.

### Spot Instance
Pricing option using spare compute capacity at discounted rates.

### AWS CLI
Command-line tool for interacting with AWS services.

### AWS SDK
Software development kits for using AWS services in applications.

### Terraform
Infrastructure as Code tool for provisioning and managing cloud resources.

### CloudWatch
Monitoring service for AWS resources and applications.

### AWS Shared Responsibility Model
Model defining security responsibilities between AWS and customers.

### IAM MFA
Multi-factor authentication for IAM users enhancing security.

### VPC Subnet
Segment of VPC IP address range for organizing resources.

### Route Table
Set of rules determining where network traffic is directed.

### VPC Peering
Networking connection between two VPCs for resource sharing.

### EC2 Auto Scaling
Service for automatically adjusting compute capacity based on demand.

### ELB (Elastic Load Balancing)
Service for distributing incoming traffic across multiple targets.

### EBS Snapshot
Point-in-time copy of EBS volume for backup and recovery.

### Cross-Region Replication
Feature for automatically copying objects to another region.

### Server-Side Encryption
S3 feature encrypting data at rest using AWS-managed or customer keys.

### Pre-signed URL
Time-limited URL granting temporary access to private S3 object.

### AWS Config
Service for assessing, auditing, and evaluating resource configurations.

### AWS CloudTrail
Service for logging API calls and user activity across AWS.

### Cost Explorer
Tool for analyzing AWS costs and usage patterns.

### AWS Budgets
Feature for setting custom cost and usage alerts.

### Placement Group
Logical grouping of instances for low-latency networking.

### Local Zone
Edge location bringing AWS services closer to end users.

### Outpost
AWS infrastructure deployed on-premises for hybrid cloud.

### GovCloud
Region designed for US government compliance requirements.
