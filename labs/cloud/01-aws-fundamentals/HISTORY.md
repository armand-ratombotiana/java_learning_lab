# History of AWS Fundamentals

## Timeline

| Year | Milestone |
|------|-----------|
| 2002 | Amazon launches internal "Amazon.com" platform services |
| 2003 | Chris Pinkham and Benjamin Black propose selling compute as a utility |
| 2004 | Initial SQS queue service launched internally |
| **2006** | **AWS publicly launches with SQS, S3, and EC2** |
| 2007 | AWS reaches 180,000 developers; IAM launches |
| 2008 | Elastic Block Store (EBS), Availability Zones, and CloudWatch launch |
| 2009 | VPC, Route 53, and Elastic Load Balancing arrive |
| 2010 | AWS introduces IAM roles; first re:Invent conference |
| 2011 | CloudFormation (IaC predecessor) launches |
| 2012 | AWS holds first re:Invent; DynamoDB released |
| 2013 | AWS GovCloud; EC2 becomes enterprise-grade |
| 2015 | AWS reaches $7.88B revenue; Lambda (serverless) launches |
| 2016 | AWS re:Invent attendance exceeds 32,000 |
| 2017 | AWS Snowball, Aurora, and SageMaker launch |
| 2018 | AWS reaches $25.6B annual revenue |
| 2020 | AWS surpasses $45B; 175+ services available |
| 2023 | AWS revenue exceeds $85B; 200+ services |
| 2024 | AWS re:Invent focuses on generative AI and Bedrock |

## Key Insight
AWS started as an internal solution to Amazon's own infrastructure scaling problems. The three original services — SQS, S3, and EC2 — were the minimum viable platform. IAM and VPC came later as the multi-tenant architecture matured and enterprise customers demanded isolation and security.

## Java on AWS History
- 2007: First AWS SDK for Java (v1.0)
- 2011: AWS SDK for Java v1.3 adds IAM support
- 2017: AWS SDK v2 for Java (reactive, non-blocking)
- 2020: AWS SDK v2 becomes GA and recommended
- 2024: AWS SDK Java v2.25+ with CRT-based HTTP client
