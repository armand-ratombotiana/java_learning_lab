# History of AWS Compute

## Timeline

| Year | Milestone |
|------|-----------|
| 2006 | EC2 launches with one instance type and Linux |
| 2008 | EC2 adds Windows support, Elastic IP, load balancers |
| 2010 | EC2 Reserved Instances; Auto Scaling launched |
| 2011 | Elastic Beanstalk (PaaS for Java, .NET, PHP, Node.js) |
| 2012 | EC2 Spot Instances (up to 90% discount) |
| 2013 | EC2 m3 instances (first SSD instance storage) |
| 2014 | **AWS Lambda** launches (compute without servers) |
| 2015 | ECS (EC2 Container Service) — Docker orchestration |
| 2016 | EC2 T2 burstable instances; Lambda adds VPC support |
| 2017 | **Fargate** (serverless containers); EKS preview |
| 2018 | **EKS** generally available; Lambda layers |
| 2019 | EC2 Nitro becomes default; Lambda Provisioned Concurrency |
| 2020 | Lambda supports graviton (ARM) — 34% better price/performance |
| 2021 | EC2 Trn1 (Trainium) for ML training |
| 2022 | Lambda SnapStart (function pre-initialization) |
| 2023 | ECS Service Connect; Lambda response streaming |
| 2024 | EC2 Capacity Blocks for ML; EKS Auto Mode |

## Key Milestones for Java
- 2014: Lambda supports Java (initially Java 8)
- 2018: Lambda Java 8+ with improved cold starts
- 2020: Lambda container support — package Java app as Docker image
- 2022: SnapStart for Java — reduce cold starts 90%
- 2023: Lambda Java 17, Java 21 runtime support
- 2024: AWS SDK for Java v2 first-class Lambda support
