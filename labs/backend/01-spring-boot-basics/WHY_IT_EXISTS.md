# Why It Exists

Before Spring Boot, Spring developers faced significant friction:

1. **Boilerplate configuration** - XML and Java config for every project
2. **Dependency management** - Manually selecting compatible library versions
3. **Embedded server setup** - Deploying WAR files to external Tomcat
4. **Environment differences** - Manually managing profiles and properties

Spring Boot solves these by providing:
- Opinionated auto-configuration
- Curated starter dependencies
- Embedded servlet containers
- Externalized configuration with profiles

Spring Boot exists to make Spring **productive out of the box** while retaining Spring's power and flexibility.
