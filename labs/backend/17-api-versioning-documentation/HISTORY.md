# History: API Versioning & Documentation

## Early Days (2000s)
REST APIs began with informal documentation. WSDL was used for SOAP but was complex. Early REST APIs often had no documentation or inline HTML docs.

## Swagger 1.0 (2011)
Tony Tam created Swagger at Reverb Technologies. It was the first specification for REST API documentation. Swagger UI provided interactive documentation.

## Swagger 2.0 (2014)
The specification was donated to the OpenAPI Initiative under Linux Foundation. Swagger 2.0 became the industry standard.

## OpenAPI 3.0 (2017)
The first version under the OpenAPI name. Major improvements:
- Request bodies separated from parameters
- Callbacks for webhooks
- Links for API chaining
- Improved security definitions

## OpenAPI 3.1 (2021)
Aligned with JSON Schema 2020-12 for better schema validation.

## SpringDoc Evolution
SpringFox was the original Spring integration for Swagger. SpringDoc emerged as a modern alternative with Spring Boot 2.x and became the standard for Spring Boot 3.x.
