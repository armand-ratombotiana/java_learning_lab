# Why Trees Matter

Trees matter because **hierarchy is everywhere** and trees enable **logarithmic-time operations**.

## Practical Impact

- **File systems**: directories (internal nodes) contain files (leaves) — the entire file system is a tree
- **Databases**: B-trees and B+ trees index billions of rows with O(log n) lookups
- **Compilers**: abstract syntax trees (ASTs) represent code structure
- **Networking**: routing tables use tree structures for IP prefix matching
- **UI**: HTML DOM is a tree; React's virtual DOM is a tree
- **AI**: decision trees, random forests, game trees (minimax)
- **Data compression**: Huffman coding uses a binary tree for optimal prefix codes

## Why Learn Trees

1. **Foundation for advanced structures**: AVL, Red-Black, B-trees, tries, segment trees
2. **Recursive thinking**: tree algorithms are the best practice for recursive problem-solving
3. **Interview essential**: tree traversals, LCA, diameter, serialization — all common topics
4. **Database internals**: understanding B-trees is crucial for query optimization
5. **Divide and conquer**: trees naturally model recursive partitioning (merge sort, quicksort)

Trees are the **bridge between linear data structures (arrays, lists) and graph theory** — they introduce concepts of depth, height, balance, and traversal strategies.
