# Exercises: Link-Cut Tree

## Basic Exercises

### Exercise 1: Implementation
Implement the core data structure from scratch. Include all basic operations and ensure all invariants are maintained.

### Exercise 2: Iterator
Create a custom iterator that traverses the structure in the correct order. Ensure it handles concurrent modification gracefully.

### Exercise 3: Serialization
Implement serialization and deserialization methods that save and restore the structure's state.

## Intermediate Exercises

### Exercise 4: Bulk Operations
Implement operations for merging two structures, splitting at a position, or performing range queries.

### Exercise 5: Performance Analysis
Write a benchmark that measures operation times for various sizes. Compare against standard library alternatives.

### Exercise 6: Visualization
Create a method that produces a text-based or graphical visualization of the structure's current state.

## Advanced Exercises

### Exercise 7: Concurrent Version
Implement a thread-safe version using read-write locks or lock-free techniques.

### Exercise 8: Persistent Variant
Modify the structure to support persistence, where old versions remain accessible after modifications.

### Exercise 9: Custom Allocator
Implement a custom memory allocator that reduces GC pressure by managing node allocation in bulk.

## Challenge Exercises

### Exercise 10: Academic Challenge
Research and implement an advanced variant from recent academic literature. Compare its performance to the standard version.
