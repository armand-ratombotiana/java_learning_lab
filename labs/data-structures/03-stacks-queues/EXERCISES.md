# Exercises: Stacks & Queues

## Basic

1. **ArrayStack** — Implement a generic stack using a dynamic array with `push`, `pop`, `peek`, `isEmpty`, `size`.

2. **CircularQueue** — Implement a fixed-capacity circular queue with `enqueue`, `dequeue`, `peek`, `isFull`, `isEmpty`.

3. **Valid parentheses** — Given a string containing `()`, `{}`, `[]`, determine if it is valid (every opening bracket has a matching closing bracket in the correct order).

4. **Queue using two stacks** — Implement a FIFO queue using two LIFO stacks.

## Intermediate

5. **MinStack** — Implement a stack that supports `push`, `pop`, `top`, and retrieving the minimum element in O(1) time.

6. **Stack using queues** — Implement a LIFO stack using two FIFO queues.

7. **Sliding window maximum** — Given an array and k, find the maximum in every sliding window of size k. O(n) using monotonic deque.

8. **Next greater element** — For each element in an array, find the next element that is greater. O(n) using monotonic stack.

9. **Priority queue** — Implement a min-heap-based priority queue with `add`, `poll`, `peek`.

## Advanced

10. **MaxStack** — Design a stack that supports `push`, `pop`, `top`, `peekMax`, and `popMax` (all O(1) except popMax O(n) or O(log n)).

11. **LRU Cache** — Design a data structure that supports `get` and `put` in O(1) average time. Use doubly linked list + HashMap.

12. **Expression evaluation** — Evaluate infix expressions with `+`, `-`, `*`, `/`, and parentheses using two stacks (operand and operator).

13. **BFS on a graph** — Implement breadth-first search using a queue to find shortest path in an unweighted graph.
