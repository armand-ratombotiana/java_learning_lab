# History of LRU Cache

## Origins

The LRU replacement policy dates back to the 1960s in operating system virtual memory management. It was formalized as a page replacement algorithm for paging systems.

## Key Developments

- **1960s**: Belady's MIN algorithm (optimal, but requires future knowledge)
- **1968**: LRU used in Multics operating system
- **1980s**: Hardware LRU caches in processors
- **1990s**: Distributed caching systems (Memcached, 2003)
- **2000s**: Redis (2009) with LRU eviction
- **2010s**: CDN caching, browser caching

## Modern Significance

Today, LRU is the default or primary caching strategy in most systems, from CPU caches to distributed web caches.
