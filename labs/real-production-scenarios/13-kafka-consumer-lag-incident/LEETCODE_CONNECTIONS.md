# Lab 13 — Kafka Consumer Lag: LeetCode Connections

**Q1: Design a Bounded Blocking Queue (LeetCode 1188)** — Kafka partitions are bounded blocking queues — producers put, consumers take.

**Q2: Design a HashMap (LeetCode 706)** — Kafka topic partitioning by key hash (key → partition assignment).

**Q3: Sliding Window Maximum (LeetCode 239)** — Consumer lag monitoring uses sliding windows to track lag trends.

**Q4: Design Hit Counter (LeetCode 362)** — Track message rates (produce rate, consume rate) per partition.

**Q5: Moving Average from Data Stream (LeetCode 346)** — Moving average of consumer lag for trend detection.

**Q6: Find Median from Data Stream (LeetCode 295)** — Monitoring P50/P99 consumer lag for alerting thresholds.

**Q7: Design a Leaderboard (LeetCode 1244)** — Consumer group rebalancing leader election (group coordinator).

**Q8: Design In-Memory File System (LeetCode 588)** — Kafka log storage as filesystem (segments, indexes, compaction).

**Q9: Task Scheduler (LeetCode 621)** — Consumer processing scheduling — coordinating message processing across consumers.

**Q10: Serialize and Deserialize Binary Tree (LeetCode 297)** — Kafka message serialization (Avro, Protobuf, JSON schema management).
