# Lab 12 — Kubernetes Pod CrashLoop: LeetCode Connections

**Q1: Design a HashMap (LeetCode 706)** — Kubernetes ConfigMaps and Secrets are key-value stores like hash maps.

**Q2: Design a File System (LeetCode 588)** — Container filesystem layers, volume mounts, overlay2 filesystem.

**Q3: LRU Cache (LeetCode 146)** — Container image layer caching — eviction policy for local image cache.

**Q4: Serialize and Deserialize Binary Tree (LeetCode 297)** — Kubernetes resource serialization (YAML/JSON), deployment manifests.

**Q5: Design In-Memory File System (LeetCode 588)** — Pod filesystem structure, emptyDir volumes, hostPath volumes.

**Q6: Implement Trie (LeetCode 208)** — Kubernetes resource hierarchy — namespace → deployment → pod → container.

**Q7: Design Bounded Blocking Queue (LeetCode 1188)** — Kubernetes scheduler queue — pods waiting for node assignment.

**Q8: Rate Limiter (LeetCode 359)** — Kubernetes resource quotas, limit ranges for namespace-level rate limiting.
