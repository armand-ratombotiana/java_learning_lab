# Lab 15 — Disaster Recovery Failover: LeetCode Connections

**Q1: Design Consistent Hashing (LeetCode — system design)** — Active-active traffic distribution across regions.

**Q2: Design a URL Shortener (LeetCode 534)** — Global load balancing, DNS-based failover, multi-region data replication.

**Q3: Design a Web Crawler (LeetCode 1242)** — Distributed processing failover — if one crawler fails, others take over.

**Q4: Design a File System (LeetCode 588)** — Multi-region file storage with replication and failover.

**Q5: Design a Key-Value Store (system design)** — Distributed database with replication, quorum, failover.

**Q6: Design a Distributed Logger (system design)** — Cross-region log aggregation for DR monitoring.

**Q7: Design a Rate Limiter (LeetCode 359)** — Rate limiting across regions with consistent state during failover.

**Q8: LRU Cache (LeetCode 146)** — Cache warming after failover — LRU helps prioritize what to cache first.

**Q9: Design a HashMap (LeetCode 706)** — Global distributed hash map for configuration, feature flags, routing tables.

**Q10: Serialize and Deserialize Binary Tree (LeetCode 297)** — Serializing DR state, configuration, and runbooks for automated recovery.
