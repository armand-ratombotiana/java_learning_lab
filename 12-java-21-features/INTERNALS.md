# Java 21 Features Internals

## Virtual Threads (Project Loom)

### Thread vs Virtual Thread Architecture

```
Platform Thread:                    Virtual Thread:
┌─────────────────────────┐       ┌─────────────────────────┐
│  OS Thread (1:1)        │       │  Virtual Thread (M:N)    │
│  - Fixed stack (~1MB)   │       │  - M virtual : N carrier│
│  - Blocking = OS block  │       │  - Dynamic stack        │
│  - Heavyweight          │       │  - Continuation-based   │
└─────────────────────────┘       └─────────────────────────┘
```

### Continuation Implementation

Virtual threads use **continuations** to suspend/resume execution without blocking OS threads:

```java
// What happens under the hood
Thread.startVirtualThread(() -> {
    try {
        blockingOperation();  // Suspends continuation
        // ... continuation resumes here
    } catch (InterruptedException e) {
        // ...
    }
});
```

### Carrier Thread Pool

Virtual threads are scheduled onto a pool of carrier threads (typically size = available processors):

```
Virtual Thread 1 ─┐
Virtual Thread 2 ─┼─→ Carrier Thread Pool (ForkJoinPool)
Virtual Thread 3 ─┘        ↓
                  ┌──────────────────┐
                  │ Carrier Thread 1 │
                  │ Carrier Thread 2 │
                  │ Carrier Thread N │
                  └──────────────────┘
```

### Memory Model

| Aspect | Platform Thread | Virtual Thread |
|--------|-----------------|----------------|
| Stack | Fixed ~1MB | 4KB - 1MB (grows/shrinks) |
| Creation | ~1MB+ | ~200 bytes |
| Context switch | OS-level | Java-level |
| Blocking | OS blocking | Continuation yield |

### Pinning and Virtual Thread Behavior

When a virtual thread calls synchronized code or blocks on a monitor, it "pins" to its carrier thread:

```java
synchronized (lock) {  // Pinning occurs here
    // Virtual thread pinned to carrier
}
```

**Pinning is NOT bad** - it's just less efficient than park/unpark. The JVM will automatically migrate virtual threads to different carriers when waiting on `LockSupport.park()` (ReentrantLock, Condition, etc.).

### JVM Implementation Details

```
HotSpot VM changes:
- Thread.startVirtualThread() creates virtual thread object
- Continuation.run() manages stack segments (StackChunk)
- Carrier thread runs ForkJoinPool worker
- park()/unpark() replaced with continuum_yield/continuum_ready
```

### Performance Characteristics

| Metric | Platform Thread | Virtual Thread |
|--------|-----------------|----------------|
| Create 10K threads | ~10 seconds | ~100ms |
| Memory for 10K | ~10GB | ~2GB |
| Context switch | ~1-2μs | ~100ns |
| Throughput (I/O bound) | Limited by threads | Scales linearly |

## Record Classes

### Bytecode Generation

Records generate specific bytecodes automatically:

```java
record Point(int x, int y) {}
```

Generates:

```
// Constructor (canonical)
public <init>(int, int) ...
  
// Accessors
public int x()
public int y()
  
// equals(), hashCode(), toString()
// static fields for each component
// equals() implementation checks same type, then field equality
```

### Record Metadata

```java
@java.lang.reflect.Record
public abstract class Point 
    extends java.lang.Record 
    implements java.io.Serializable {
    // Components stored as final fields
    private final int x;
    private final int y;
    
    // Canonical constructor
    // Accessor methods
    // equals() - component-by-component comparison
    // hashCode() - Objects.hash(x, y)
    // toString() - "Point[x=1, y=2]"
}
```

### Record Component Accessors

```java
// Generated bytecode for accessor
public final int x();
  descriptor: ()I
  flags: (0x0011) ACC_PUBLIC, ACC_FINAL
  Code:
    aload_0
    getfield      #2          // Field x:I
    ireturn
```

### Canonical Constructor

The canonical constructor is the one matching the record components:

```java
record Person(String name, int age) {
    // Canonical constructor (auto-generated if not provided)
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

### Compact Constructor

```java
record Point(int x, int y) {
    // Compact constructor - no parameters
    public Point {
        if (x < 0 || y < 0) 
            throw new IllegalArgumentException();
    }
}
```

Compacts to the same signature as the implicit canonical constructor.

### Record Serialization

Records use standard Java serialization with special handling:

```java
// Serializable implementation
record User(String name, int id) implements Serializable {}

// Serialized form is like:
// ObjectStreamField[] = { "name", "id" }
// readObject/writeObject handle canonical form
```

### Pattern Matching for Records

```java
// Destructive pattern matching (Java 16+)
if (obj instanceof Point(int x, int y)) {
    System.out.println(x + y);
}

// Nested patterns (Java 21)
if (obj instanceof RecordContainer(Point(int x, int y), String label)) {
    System.out.println(label + ": " + x + y);
}
```

### Record and Reflection

```java
record Point(int x, int y) {}

var clazz = Point.class;
// Check if record
clazz.isRecord()  // true

// Get component types
clazz.getRecordComponents()  // [x: int, y: int]

// Get accessor
clazz.getMethod("x")  // public final int x()
```

## Pattern Matching for Switch (Java 21)

### Type Patterns

```java
Object obj = "hello";
String result = switch(obj) {
    case Integer i -> "int: " + i;
    case String s -> "string: " + s.length();
    case null -> "null";
    default -> "unknown";
};
```

### Guards

```java
sealed interface Animal permits Dog, Cat {}
record Dog(String name) implements Animal {}
record Cat(int lives) implements Animal {}

String check(Animal a) {
    return switch(a) {
        case Dog d && d.name().startsWith("R") -> "Rex!";
        case Dog d -> "dog: " + d.name();
        case Cat c && c.lives() == 9 -> "lucky cat";
        case Cat c -> "cat with " + c.lives() + " lives";
    };
}
```

### Exhaustiveness Checking

```java
sealed interface Shape permits Circle, Rectangle {}

String evaluate(Shape s) {
    return switch(s) {
        case Circle c -> "circle";
        case Rectangle r -> "rect";
        // Compiler error if not exhaustive
    };
}
```

## Sequenced Collections (Java 21)

```java
// New interface hierarchy
interface SequencedCollection<E> extends Collection<E> {
    E getFirst();
    E getLast();
    void addFirst(E);
    void addLast(E);
    SequencedCollection<E> reversed();

List<String> list = new ArrayList<>();
list.addFirst("first");
list.reversed();  // Returns reversed view
```

## String Templates (Preview in Java 21)

```java
import java.lang.StringTemplate;

// STR template processor
String name = "Alice";
int age = 30;
String message = STR."Hello \{name}, you are \{age}";

/*
 * Generated code similar to:
 * new StringTemplate(
 *     () -> "Hello ",
 *     () -> name,
 *     ", you are ",
 *     () -> age
 * )
 */
```

### Template Processors

```java
// RAW - returns raw template fragments
StringTemplate.Raw raw = RAW."Name: \{name}";

// FMT - formatted string templates
String formatted = FMT."%-8s\t%-8s\n\{name}\{age}";

// Custom processor
var processor = StringTemplate.Processor.of(
    (StringTemplate st) -> st.fragments().stream()
        .map(s -> s.toUpperCase())
        .collect(Collectors.joining())
);
```
